⚠️ Récapitulatif — Erreurs clés à éviter

❌

Confondre = (assignation) et == (comparaison)

Python
if age = 18:    # ❌ SyntaxError
if age == 18:   # ✅
❌

Accéder à une clé de dict inexistante avec []

Python
d["john"]                  # ❌ KeyError si "john" absent
d.get("john", "Default")   # ✅ retourne "Default" si absent
❌

Oublier le f devant une f-string

Python
"Hi, my name is {name}"   # ❌ affiche littéralement "{name}"
f"Hi, my name is {name}"  # ✅
❌

Modifier une liste pendant une boucle for

Python
for word in words:
    words.remove(word)     # ❌ comportement imprévisible
for word in words.copy():  # ✅ itérer sur une copie
    words.remove(word)
❌

Utiliser and/or au lieu de &/| dans un boolean indexing Pandas

Python
df[df["a"] > 1 and df["b"] < 5]    # ❌ ValueError
df[(df["a"] > 1) & (df["b"] < 5)]  # ✅
❌

Lancer make sans avoir testé son code avant

Bash
## ❌ make directement → erreurs difficiles à interpréter
python example.py  # ✅ toujours tester avec python d'abord

🧭 Introduction

Python est aujourd'hui le langage numéro 1 en data science. Pourquoi ? Parce qu'il est lisible, puissant, et entouré d'un écosystème de bibliothèques extraordinaire — notamment NumPy (calculs numériques), Pandas (tableaux de données) et Matplotlib (visualisation). Mais avant de toucher à ces outils avancés, il faut maîtriser les bases du langage lui-même.

A retenir absolument

  • Python est interprété : le code s'exécute ligne par ligne, pas besoin de compilation.
  • Tout est objet : les nombres, les textes, les listes… tout a des méthodes attachées.
  • L'indentation (les espaces en début de ligne) est obligatoire et structurante.
  • Une variable est une boîte qui contient une valeur — on peut changer ce qu'elle contient.
  • Les fonctions permettent de réutiliser du code sans le réécrire.

1) Python — Vue d'ensemble

Python a été créé en 1991 par Guido van Rossum. Ses grandes caractéristiques :

  • Multi-plateforme : tourne sur Windows, Mac, Linux.
  • Interprété : pas besoin de compiler, on exécute directement.
  • Typage dynamique : pas besoin de déclarer le type d'une variable.
  • Garbage collected : Python gère lui-même la mémoire, tu n'as pas à t'en préoccuper.
  • Riche en bibliothèques : des milliers de modules disponibles.

Le REPL — ton terrain de jeu interactif

REPL signifie Read, Eval, Print, Loop : tu tapes une ligne, Python l'évalue et affiche le résultat, puis attend la suivante. C'est parfait pour tester des idées rapidement.

Pour lancer le REPL enrichi (avec coloration syntaxique et autocomplétion) :

Bash
ipython   # lance un terminal Python interactif
💡

Le REPL est ton meilleur ami pour expérimenter. Avant d'écrire un script complet, teste toujours tes idées dans ipython !

2) Types built-in simples

Les "types" en Python, c'est la nature d'une donnée : est-ce un nombre entier ? Un texte ? Un vrai/faux ?

2.1 Numériques : int et float

Python
type(42)    # <class 'int'>   — nombre entier
type(3.14)  # <class 'float'> — nombre décimal
4 + 9.5     # 13.5            — int + float = float
abs(-5)     # 5               — valeur absolue

La fonction type() te dit de quel type est une valeur. abs() retourne la valeur absolue (toujours positive).

🧠

En data science, tu manipuleras des milliers de nombres. Savoir distinguer int et float évite des bugs subtils — par exemple, la division entière 7 // 2 donne 3, pas 3.5.

2.2 Booléens (bool) — les valeurs vrai/faux

Un booléen ne peut prendre que deux valeurs : True ou False. C'est la base de tout raisonnement logique.

Python
type(True)      # <class 'bool'>
True and False  # False — les deux doivent être vrais
True or False   # True  — au moins l'un doit être vrai
not True        # False — inverse la valeur

Les valeurs "falsy" — Python les considère comme False dans une condition :

ValeurType
Noneabsence de valeur
Falsebooléen
0, 0.0zéros numériques
"", [], {}, ()séquences vides

Comparaisons — pour tester des conditions :

Python
5 < 3       # False — est-ce que 5 est strictement inférieur à 3 ?
4 <= 4      # True  — est-ce que 4 est inférieur ou égal à 4 ?
"a" == "b"  # False — est-ce que "a" est égal à "b" ? (double égal !)
"a" != "b"  # True  — est-ce que "a" est différent de "b" ?
⚠️

Ne confonds jamais = (assignation — tu donnes une valeur) et == (comparaison — tu poses une question). age = 18 met 18 dans la variable. age == 18 demande "est-ce que age vaut 18 ?"

2.3 Chaînes de caractères (str)

Une chaîne, c'est du texte. Elle s'écrit entre guillemets simples ou doubles.

Python
"boris"                   # une chaîne de caractères
"boris".capitalize()      # "Boris" — met la première lettre en majuscule
"   boris  ".strip()      # "boris" — supprime les espaces en début et fin
type(int("42"))           # <class 'int'> — convertit la chaîne "42" en nombre 42

Python offre de nombreuses méthodes sur les chaînes. Quelques exemples utiles :

Python
"bonjour".upper()         # "BONJOUR" — tout en majuscules
"BONJOUR".lower()         # "bonjour" — tout en minuscules
"data science".replace("data", "big data")  # "big data science"
len("hello")              # 5 — nombre de caractères
💡

Une "méthode" se place après un point : valeur.methode(). C'est comme dire à l'objet "fais quelque chose avec toi-même".

3) Types built-in avancés

3.1 La liste (list) — une séquence ordonnée

Une liste est une collection ordonnée de valeurs. Elle peut contenir des éléments de types différents.

Python
["apple", "cherries", "banana"]    # une liste de 3 chaînes

len(["apple", "cherries", "banana"])           # 3    — combien d'éléments ?
"banana" in ["apple", "cherries", "banana"]    # True — est-ce que "banana" est dans la liste ?

Les listes sont mutables : on peut les modifier après leur création (ajouter, supprimer, modifier des éléments).

3.2 Le tuple (tuple) — une liste non modifiable

Python
(1, 2, 3)   # un tuple — comme une liste, mais immuable (on ne peut pas le modifier)
🧠

Utilise un tuple quand tu as des données qui ne doivent pas changer — par exemple, les coordonnées GPS d'une ville : (48.8566, 2.3522) pour Paris.

3.3 Le range — une séquence de nombres

Python
range(5)   # représente 0, 1, 2, 3, 4 — souvent utilisé dans les boucles
list(range(5))   # [0, 1, 2, 3, 4] — convertit le range en liste visible

3.4 Le dictionnaire (dict) — clé → valeur

Un dictionnaire associe des clés à des valeurs. C'est comme un vrai dictionnaire : tu cherches un mot (clé) pour trouver sa définition (valeur).

Python
{"city": "Paris", "population": 2_141_000}   # dict avec 2 paires clé-valeur

len({"city": "Paris", "population": 2_141_000})       # 2    — nombre de paires
"city" in {"city": "Paris", "population": 2_141_000}  # True — la clé existe ?
💡

Les underscores dans les nombres (2_141_000) sont purement visuels — Python les ignore. C'est juste plus lisible que 2141000.

4) Variables

Une variable est une étiquette que tu colles sur une valeur pour la retrouver plus tard.

Python
name = "John"   # on colle l'étiquette "name" sur la valeur "John"
print(name)     # affiche : John

Tu peux réassigner une variable — l'étiquette se déplace vers la nouvelle valeur :

Python
name = "Paul"   # l'étiquette "name" pointe maintenant vers "Paul"
print(name)     # affiche : Paul

Tu peux aussi combiner des variables :

Python
first_name = "John"              # variable prénom
last_name  = "Lennon"            # variable nom

full_name = first_name + " " + last_name  # concaténation avec +
print(full_name)                 # affiche : John Lennon

Les f-strings — interpolation propre

Au lieu de concaténer avec +, les f-strings permettent d'insérer des variables directement dans une chaîne. Plus lisible, moins d'erreurs.

Python
sentence = f"Hi, my name is {first_name} {last_name}"
# Le f devant les guillemets active l'interpolation
# {first_name} est remplacé par la valeur de la variable
print(sentence)   # affiche : Hi, my name is John Lennon
⚠️

Oublie le f devant la chaîne et Python affichera littéralement {first_name} au lieu de la valeur. C'est une erreur très courante !

5) Opérations sur les listes

Accéder aux éléments — les index

Les index commencent à 0 (pas 1). C'est une convention universelle en informatique.

Python
#            0       1       2       ← index de chaque élément
beatles = ["john", "paul", "ringo"]

beatles[0]   # "john"  — le premier élément (index 0)
beatles[2]   # "ringo" — le troisième élément (index 2)
beatles[-1]  # "ringo" — le dernier élément (index -1 = depuis la fin)

Créer — ajouter un élément

Python
beatles.append("george")   # ajoute "george" à la fin, modifie la liste en place
print(beatles)
# ['john', 'paul', 'ringo', 'george']

Modifier — changer un élément

Python
beatles[1] = "Paul"   # remplace l'élément à l'index 1 par "Paul"
print(beatles)
# ['john', 'Paul', 'ringo', 'george']

Extraire une portion — le slice

Le slice (tranche) extrait une sous-liste. Syntaxe : liste[debut:fin] — la fin est exclue.

Python
beatles[1:3]   # extrait de l'index 1 (inclus) à l'index 3 (exclu)
# ['Paul', 'ringo']
$$\text{slice}[1:3] \Rightarrow \text{index } 1 \text{ et } 2 \text{ (3 exclu)}$$

Supprimer — effacer un élément

Python
del beatles[1]   # supprime l'élément à l'index 1
print(beatles)
# ['john', 'ringo', 'george']
✅

Récapitulatif des opérations sur les listes : liste[i] pour lire, .append() pour ajouter, liste[i] = val pour modifier, del liste[i] pour supprimer, liste[a:b] pour trancher.

6) Opérations sur les dictionnaires

Créer et lire

Python
instruments = {"john": "guitar", "paul": "bass"}
# clés : "john", "paul"   valeurs : "guitar", "bass"

instruments["john"]   # "guitar" — accès par la clé

Ajouter ou modifier

Python
instruments["ringo"] = "drums"   # si la clé n'existe pas, elle est créée
print(instruments)
# {'john': 'guitar', 'paul': 'bass', 'ringo': 'drums'}

instruments["john"] = "rhythm guitar"   # si elle existe, elle est mise à jour

Supprimer

Python
del instruments["john"]   # supprime la paire clé-valeur
print(instruments)
# {'paul': 'bass', 'ringo': 'drums'}

Accès sécurisé avec .get()

Si tu accèdes à une clé qui n'existe pas avec [], Python lève une KeyError. La méthode .get() permet d'éviter ce crash en fournissant une valeur par défaut :

Python
instruments["john"]                        # KeyError: 'john' — clé absente !
instruments.get("john", "no instrument")   # "no instrument" — valeur de secours
⚠️

En data science, tu ne sais pas toujours quelles clés existent dans tes données. Utilise .get() par défaut pour éviter les crashes inattendus.

7) Control Flow — diriger l'exécution

Le "control flow" (flux de contrôle) permet au programme de prendre des décisions et répéter des actions.

7.1 if / elif / else — les conditions

C'est la structure "si... sinon si... sinon...". Python évalue chaque condition dans l'ordre et exécute le premier bloc vrai.

Python
age = int(input("How old are you?"))  # demande l'âge à l'utilisateur et le convertit en int

if age >= 21:                          # si age est supérieur ou égal à 21
    print("You can become president")  # ← ce bloc s'exécute si la condition est vraie
elif age >= 18:                        # sinon, si age est supérieur ou égal à 18
    print("You can vote")              # ← ce bloc s'exécute dans ce cas
else:                                  # sinon (tous les autres cas)
    print("Be patient!")               # ← ce bloc s'exécute en dernier recours
🧠

L'indentation (4 espaces) est obligatoire — c'est elle qui délimite les blocs. Pas de {} comme en JavaScript ou Java.

7.2 for — répéter pour chaque élément

La boucle for parcourt chaque élément d'une collection, un par un.

Python
words = ["cat", "wolf", "beetle"]   # notre liste de départ

for word in words:                  # pour chaque mot dans words...
    print(word.upper())             # ...affiche le mot en majuscules
# CAT
# WOLF
# BEETLE

7.3 List comprehension — la boucle en une ligne

Une list comprehension est une syntaxe Python concise pour créer une nouvelle liste à partir d'une autre. C'est l'équivalent d'une boucle for condensée en une ligne.

Python
# Version longue (boucle for classique)
uppercase_words = []
for word in words:
    uppercase_words.append(word.upper())

# Version courte (list comprehension) — même résultat
uppercase_words = [word.upper() for word in words]
# ['CAT', 'WOLF', 'BEETLE']

La structure générale est : [expression for element in liste]

$$[\underbrace{word.upper()}_{\text{transformation}} \text{ for } \underbrace{word}_{\text{variable}} \text{ in } \underbrace{words}_{\text{liste source}}]$$
💡

La list comprehension est très utilisée en data science pour transformer des colonnes ou filtrer des données. Entraîne-toi à la lire de gauche à droite : "donne-moi [transformation] pour chaque [élément] dans [liste]".

7.4 enumerate — boucle avec index

Quand tu as besoin à la fois de la valeur et de sa position dans la liste :

Python
for index, word in enumerate(words):   # enumerate fournit (index, valeur)
    print(index, word)                 # affiche l'index puis le mot
# 0 cat
# 1 wolf
# 2 beetle

7.5 for sur un dictionnaire

Pour parcourir les paires clé-valeur d'un dictionnaire, utilise .items() :

Python
instruments = {"paul": "bass", "ringo": "drums"}

for beatle, instrument in instruments.items():              # décompose chaque paire
    print(f"{beatle.capitalize()} plays the {instrument}")  # affiche une phrase
# Paul plays the bass
# Ringo plays the drums

7.6 while — boucle conditionnelle

La boucle while répète un bloc tant qu'une condition est vraie. Attention à ne pas créer une boucle infinie !

Python
i = 1           # compteur initialisé à 1

while i <= 4:   # tant que i est inférieur ou égal à 4...
    print(i)    # affiche la valeur de i
    i = i + 1   # incrémente i (IMPORTANT : sans ça, boucle infinie !)
# 1
# 2
# 3
# 4
⚠️

Oublie i = i + 1 et ton programme tourne à l'infini. Pour arrêter un programme bloqué dans le terminal : Ctrl + C.

8) Fonctions

Une fonction, c'est un bloc de code réutilisable auquel on donne un nom. Au lieu de réécrire la même logique plusieurs fois, on l'encapsule dans une fonction qu'on appelle quand on en a besoin.

8.1 Fonction simple

Python
def is_even(number):           # def = définition, is_even = nom, number = paramètre
    return number % 2 == 0    # retourne True si number est pair (reste de division = 0)
  • def : mot-clé pour définir une fonction
  • is_even : le nom qu'on lui donne (convention snake_case)
  • number : le paramètre (la donnée qu'on lui passe)
  • return : ce que la fonction renvoie

Appel de la fonction :

Python
result = is_even(42)   # on appelle is_even avec l'argument 42
print(result)          # True — 42 est pair (42 % 2 = 0)

result = is_even(7)    # on appelle is_even avec l'argument 7
print(result)          # False — 7 est impair (7 % 2 = 1)

L'opérateur % (modulo) donne le reste de la division entière :

$$42 \div 2 = 21 \text{ reste } 0 \Rightarrow \text{pair}$$
$$7 \div 2 = 3 \text{ reste } 1 \Rightarrow \text{impair}$$

8.2 Argument avec valeur par défaut (keyword argument)

On peut donner une valeur par défaut à un paramètre. Si l'appelant ne précise pas cet argument, la valeur par défaut est utilisée.

Python
def is_odd(number=0):          # number vaut 0 par défaut si non précisé
    return number % 2 == 1    # retourne True si number est impair

is_odd(number=1)   # True  — on passe explicitement 1
is_odd()           # False — on utilise la valeur par défaut 0

8.3 Mélanger arguments positionnels et nommés

Python
def full_name(first_name, last_name, capitalize=False):
    # first_name, last_name : arguments obligatoires (positionnels)
    # capitalize : argument optionnel avec valeur par défaut False
    if capitalize:
        return f"{first_name.capitalize()} {last_name.capitalize()}"
    else:
        return f"{first_name} {last_name}"

print(full_name("john", "lennon"))                       # john lennon
print(full_name("ringo", "starr", capitalize=True))      # Ringo Starr
✅

Règle : les arguments sans valeur par défaut (obligatoires) se placent avant ceux avec valeur par défaut (optionnels). Sinon Python retourne une erreur de syntaxe.

🧠

En data science, tu écriras souvent des fonctions pour nettoyer des données, calculer des métriques ou transformer des colonnes. Bien structurer ses fonctions rend le code maintenable et testable.

9) Modules

Quand ton projet grossit, il n'est plus raisonnable de tout mettre dans un seul fichier. On découpe le code en modules — des fichiers Python séparés, importables les uns dans les autres.

Structure d'un mini-projet

Bash
mkdir project && cd project   # crée le dossier du projet
touch app.py                  # fichier principal
mkdir helpers                 # dossier pour les fonctions utilitaires
touch helpers/__init__.py     # fichier qui fait du dossier un module Python

Définir une fonction dans un module

Python
# helpers/__init__.py
import random    # module standard Python pour l'aléatoire
import string    # module standard Python pour les caractères

def generate_password(size):
    """Generate a random lowercase ASCII password of given size"""
    # string.ascii_lowercase = "abcdefghijklmnopqrstuvwxyz"
    letters = string.ascii_lowercase
    # pour chaque position (0 à size-1), choisit une lettre au hasard
    return "".join(random.choice(letters) for i in range(size))

Importer et utiliser le module

Python
# app.py
from helpers import generate_password   # importe la fonction depuis helpers

print(generate_password(16))            # génère et affiche un mot de passe de 16 caractères
Bash
python app.py   # lance le script depuis le terminal
# Exemple de sortie : xkqpzmvtnbwjrlcs
💡

Le fichier __init__.py (avec deux underscores de chaque côté) est le "point d'entrée" d'un module-dossier. Python le cherche automatiquement quand tu importes le dossier.

A. Concepts du cours

NumPy : pourquoi ndarray bat list

Une liste Python est une suite de pointeurs vers des objets dispersés en mémoire. Calculer la somme de 1 million d'entiers prend ~50 ms parce que pour chaque addition, Python doit :

  1. Déréférencer le pointeur
  2. Vérifier le type
  3. Appeler la méthode __add__
  4. Créer un nouvel objet

Un numpy.ndarray est un bloc contigu de bytes en mémoire avec un type homogène (int32, float64, etc.). La même opération prend ~1 ms — un facteur 50. Pourquoi ?

  1. Homogénéité : pas de vérification de type à chaque élément.
  2. Contiguïté mémoire : maximise l'utilisation du cache CPU.
  3. Code C compilé : la boucle s'exécute en machine, sans l'interpréteur Python.
  4. SIMD : les CPUs modernes traitent 4-8 valeurs par cycle d'horloge.
Python
import numpy as np

# Liste Python
%timeit sum(range(1_000_000))                   # ~50 ms

# NumPy
arr = np.arange(1_000_000)
%timeit arr.sum()                                # ~1 ms — 50x plus rapide
💡

Analogie : une list Python est un train où chaque wagon est ailleurs et qu'il faut aller chercher un par un. Un ndarray est un train collé wagon par wagon — vous parcourez tout d'un trait, et le moteur (CPU) tourne à plein régime sans freiner.

Vectorisation et broadcasting

La règle d'or : une boucle Python sur un ndarray est presque toujours un bug de performance. Toute opération sur ndarray devrait s'exprimer sans boucle.

Python
# ❌ Mauvais : boucle Python explicite
result = np.zeros(len(arr))
for i in range(len(arr)):
    result[i] = arr[i] ** 2 + 3

# ✓ Bon : vectorisé, ~50x plus rapide
result = arr ** 2 + 3

Le broadcasting étend automatiquement les shapes pour permettre des opérations entre arrays de tailles différentes :

Python
A = np.array([[1, 2, 3],
              [4, 5, 6]])           # shape (2, 3)
v = np.array([10, 20, 30])          # shape (3,)

A + v
# array([[11, 22, 33],
#        [14, 25, 36]])

Explication en français : NumPy a "étiré" virtuellement v pour qu'il colle à A. Aucune copie n'est faite : c'est gratuit en mémoire et en temps. Chaque ligne de A reçoit v en addition élément par élément.

Règles de broadcasting (à connaître par cœur) :

  1. Les shapes sont alignées à droite (last axis first).
  2. À chaque axe, les dimensions doivent être égales ou égales à 1 (auquel cas la dimension de 1 est étirée).
  3. Si les shapes ont un nombre d'axes différent, la plus petite est complétée par des 1 à gauche.
Python
A.shape  # (2, 3)
v.shape  # (3,)  → traité comme (1, 3) → étiré à (2, 3)
⚠️

Piège : np.zeros(10) + np.zeros(5) lève une erreur (shapes incompatibles), mais np.zeros(10) + 0 marche (le scalaire est broadcastable à n'importe quoi). Toujours vérifier arr.shape quand un broadcast échoue.

Indexation : loc, iloc, booléenne

Trois façons d'accéder à des sous-ensembles :

Python
# Par POSITION entière (iloc)

⚠️

Question d'entretien : pourquoi df.loc[0:5] renvoie-t-il 6 lignes alors que df.iloc[0:5] en renvoie 5 ? Réponse : loc utilise les labels, donc il interprète 0:5 comme "du label 0 au label 5 inclus". iloc utilise les positions et suit la convention Python (borne sup exclue). Cette incohérence est l'une des verrues historiques de Pandas — à mémoriser.

B. Compléments avancés

Le dtype et son impact mémoire

Pandas/NumPy stocke chaque colonne avec un type précis. Un mauvais dtype gonfle inutilement la mémoire.

Python
df.memory_usage(deep=True)
# age      int64    (8 bytes per row)
# price    float64  (8 bytes per row)
# city     object   (50+ bytes per row — pointeur + string !)

Optimisations courantes :

AvantAprèsGain
int64int32 ou int162x à 4x
float64float322x
object (string)category (si peu de valeurs uniques)5x à 100x
objectstring[pyarrow] (pandas 2.0+)2x
Python
# Conversion explicite
df["category"] = df["category"].astype("category")
df["small_int"] = df["small_int"].astype("int16")

# Lecture optimisée directement
df = pd.read_csv(
    "data.csv",
    dtype={"id": "int32", "category": "category"},
)
👉

Bonne pratique : sur un dataset de 10 colonnes, convertir les colonnes string à faible cardinalité en category divise souvent la mémoire totale par 5. Sur 10 Go de données, c'est 8 Go de RAM gagnés gratuitement.

Vue vs copie : le SettingWithCopyWarning

Le warning le plus craint des débutants pandas. Origine : certaines opérations renvoient une vue (référence vers les données originales), d'autres une copie. Si vous modifiez une vue en pensant que c'est une copie, vous modifiez aussi l'original — ou pire, votre modification n'a aucun effet.

Python
# ❌ Anti-pattern qui déclenche le warning
filtered = df[df["age"] > 30]
filtered["status"] = "senior"   # Ambigu : modifie filtered ou df ?

Solutions canoniques :

Python
# Option 1 : copie explicite
filtered = df[df["age"] > 30].copy()
filtered["status"] = "senior"

# Option 2 : opération atomique sur df (sans créer de variable intermédiaire)
df.loc[df["age"] > 30, "status"] = "senior"

Règle : si vous voulez modifier le DataFrame d'origine, utilisez df.loc[...] = ... en une seule expression. Si vous voulez un sous-DataFrame indépendant, faites .copy().

Pandas vs Polars vs DuckDB

Pandas a 15 ans et beaucoup de verrues historiques. Trois alternatives modernes méritent d'être connues :

OutilForcesQuand l'utiliser
PandasÉcosystème massif, interop sklearn, doc partoutDatasets < 1 Go, code partagé en équipe non-experts
Polars5-30x plus rapide, syntaxe expressive, lazy executionDatasets 1 Go - 100 Go, code performance-critique
DuckDBSQL natif, in-process, gère les fichiers Parquet géantsAnalyses ad-hoc complexes, joins de plusieurs tables
Dask / SparkTraitement distribuéDatasets > 100 Go, cluster

Polars en action — même opération, plus rapide et plus expressive :

Python
import polars as pl

df = pl.read_csv("data.csv")
result = (
    df.lazy()                           # Lazy : rien n'est exécuté encore
    .filter(pl.col("age") > 30)
    .group_by("city")
    .agg(pl.col("amount").sum())
    .collect()                          # Déclenche l'exécution optimisée
)
💡

Analogie : Pandas est à Polars ce que Python est à Rust. Pandas est plus accessible, plus permissif, mais plus lent et plus piégeant. Polars vous force à exprimer vos opérations de manière explicite et lazy, et vous récompense par une vitesse parfois 30x supérieure. Pour un projet personnel, Pandas est très bien. Pour un pipeline de prod qui tourne toutes les heures, Polars peut justifier la migration.

C. Questions de compréhension

Q1 : Vous avez une boucle for i in range(len(df)): df.loc[i, 'col'] = f(df.loc[i, 'other']). Pourquoi est-ce horrible et comment réécrire ?

💡 Voir l'indice

Vectorisation.

✅ Voir la réponse

C'est horrible pour trois raisons :

  1. Performance : chaque df.loc[i, ...] fait un lookup dans l'index, ce qui est ~100x plus lent qu'un accès vectorisé. Sur 1 million de lignes, vous passez de quelques millisecondes à plusieurs minutes.
  2. Lisibilité : on ne voit pas immédiatement ce que la boucle fait.
  3. SettingWithCopyWarning : risqué selon comment df a été créé.

La réécriture canonique :

Python
# Si f est vectorisable (opérations NumPy/Pandas)
df['col'] = f(df['other'])              # Le plus rapide, en une ligne

# Si f est non-vectorisable (fonction Python custom)
df['col'] = df['other'].apply(f)        # Encore ~10x plus rapide que la boucle

Le test mental : "puis-je remplacer la boucle par une opération sur la colonne entière ?" Si oui, faites-le. Si la fonction est vraiment non-vectorisable et que c'est lent, passez à numba ou pl.col(...).map_elements(f) en Polars.

Q2 : Quelle est la différence entre df.merge(other), df.join(other), et pd.concat([df, other]) ?

💡 Voir l'indice

Axe et clé.

✅ Voir la réponse
  • pd.concat : empile (par défaut sur l'axe 0 = lignes) ou colle côte à côte (axe 1 = colonnes), aligné par index. Aucune notion de clé.
  • df.merge(other, on='key') : équivalent du SQL JOIN. Aligne par valeurs d'une colonne commune, supporte how='inner|left|right|outer'. C'est le plus utilisé.
  • df.join(other) : raccourci de merge qui aligne par index par défaut — utile quand les deux DataFrames partagent le même index.

Règles d'or :

  • concat quand vous accumulez des chunks de même schéma (ex : lire plusieurs CSV et les coller).
  • merge quand vous combinez des tables sur une clé métier (ex : users + orders).
  • join quand vous fusionnez par index (cas plus rare).

Exemple merge :

Python
users.merge(orders, on='user_id', how='left')
# Équivalent SQL : SELECT * FROM users LEFT JOIN orders ON users.id = orders.user_id

Q3 : Vous lisez un CSV de 5 Go avec pd.read_csv et votre RAM explose. Trois techniques pour gérer ça ?

💡 Voir l'indice

Streaming, dtypes, formats.

✅ Voir la réponse

Technique 1 — Chunking : lire le fichier par morceaux.

Python
total = 0
for chunk in pd.read_csv("huge.csv", chunksize=10000):
    total += chunk['amount'].sum()    # Agréger en streaming

Technique 2 — Dtypes explicites : spécifier les types au moment de la lecture plutôt que de laisser Pandas tout deviner en int64/object qui sont gourmands.

Python
df = pd.read_csv(
    "data.csv",
    dtype={"id": "int32", "name": "category"},
    usecols=["id", "name", "amount"],   # Ne lire que les colonnes nécessaires
)
# Divise la mémoire par 5-10

Technique 3 — Changer de format : convertir en Parquet une fois pour toutes.

Python
df = pd.read_csv("huge.csv")
df.to_parquet("huge.parquet")   # 5-10x plus petit, colonnaire

Parquet compresse 5-10x mieux et lit colonne par colonne — donc lire 3 colonnes sur 50 ne charge que ces 3 colonnes.

Encore mieux : utilisez DuckDB qui sait requêter directement un fichier Parquet sans le charger en RAM :

Python
import duckdb
duckdb.sql("SELECT city, AVG(price) FROM 'data.parquet' GROUP BY city").df()

Sur des datasets > 1 Go, Parquet + DuckDB est ~100x plus efficace que CSV + Pandas.
