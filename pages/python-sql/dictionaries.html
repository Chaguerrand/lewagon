Dictionaries

Comprendre ce qu'est un **dictionnaire** (`dict`) en Python, quand l'utiliser, et maîtriser les opérations courantes (création, lecture, écriture, suppression, parcours).

🎯 Notions cles a retenir
Hash map
Structure clé→valeur à accès O(1) en moyenne.
Clé hashable
Clé immuable (str, int, tuple) — listes interdites comme clés.
`get(key, default)`
Lecture sans `KeyError` si la clé manque.
Dict comprehension
`{k: v for k, v in …}` pour construire un dict en une ligne.

📋 Cheatsheet — Dictionnaires

Python
## ── CRÉER ────────────────────────────────────────────
d = {}                                  # dict vide
d = {"nom": "Alice", "age": 30}         # dict avec valeurs initiales

## ── LIRE ─────────────────────────────────────────────
d["nom"]                        # "Alice" — KeyError si clé absente
d.get("nom")                    # "Alice" — None si clé absente (pas d'erreur)
d.get("ville")                  # None
d.get("ville", "inconnue")      # "inconnue" (valeur par défaut)

## ── ÉCRIRE / METTRE À JOUR ───────────────────────────
d["ville"] = "Paris"            # ajoute une nouvelle clé
d["age"] = 31                   # modifie une valeur existante (même syntaxe)
d.update({"age": 32, "pays": "France"})  # met à jour plusieurs clés à la fois

## ── SUPPRIMER ────────────────────────────────────────
del d["ville"]                  # supprime la clé, ne retourne rien
d.pop("age")                    # supprime et retourne la valeur associée

## ── INFOS ────────────────────────────────────────────
len(d)                          # nombre de clés dans le dict
"nom" in d                      # True si la clé existe, False sinon

## ── PARCOURIR ────────────────────────────────────────
for key in d:                   # itère sur les clés
    print(key)

for value in d.values():        # itère sur les valeurs
    print(value)

for key, value in d.items():    # itère sur les paires clé/valeur
    print(f"{key} → {value}")

## ── DICT COMPREHENSION ───────────────────────────────
{k: v*2 for k, v in d.items()}            # double chaque valeur
{k: v for k, v in zip(keys, values)}      # construit un dict depuis deux listes

⚠️ Récapitulatif — Erreurs clés à éviter

❌

Accéder à une clé inexistante avec []

Python
london = {"country": "England"}
london["population"]      # ❌ KeyError
london.get("population")  # ✅ retourne None
❌

Utiliser une valeur mutable comme clé

Python
d = {[1, 2]: "valeur"}  # ❌ TypeError: unhashable type 'list'
d = {(1, 2): "valeur"}  # ✅ les tuples sont hashables
❌

Oublier que les clés sont uniques — une assignation écrase

Python
d = {"age": 20, "age": 30}  # ❌ la clé "age": 20 est silencieusement écrasée
print(d)  # {"age": 30}
❌

Itérer sur .items() sans déstructurer

Python
for item in d.items():
    print(item[0], item[1])  # ❌ fonctionne mais peu lisible

for key, value in d.items():  # ✅
    print(key, value)
❌

Modifier un dict pendant qu'on itère dessus

Python
for key in d:
    del d[key]  # ❌ RuntimeError: dictionary changed size during iteration
## ✅ Itère sur une copie : for key in list(d.keys()):

1) Pourquoi des dictionnaires ?

Quand tu as deux listes "parallèles" (ex : noms et âges), tu dois gérer des index, ce qui devient fragile et difficile à maintenir.

Exemple (moins pratique)

Python
students = ["Peter", "Mary", "George", "Emma"]
student_ages = [24, 25, 22, 20]

for index, student in enumerate(students):
	age = student_ages[index]
	print(f"- {student} ({age} years old)")

Ce qu'on veut faire

Accéder directement à la valeur via une clé.

Python
print(students_age["Peter"])

Solution : un dict

Python
students_age = {
	"Peter": 24,
	"Mary": 25,
	"George": 22,
	"Emma": 20,
}

2) dict : définition

Un dictionnaire est une collection de paires clé → valeur.

  • Les clés sont uniques.
  • L'accès par clé est rapide et lisible.

🧭 Introduction

Jusqu'ici, pour stocker plusieurs informations, tu utilisais probablement des listes. Mais les listes ont une limite : pour retrouver une valeur, il faut connaître sa position (index 0, 1, 2…). C'est vite inconfortable quand les données ont un sens naturel de type "nom → valeur".

Les dictionnaires (dict) résolvent ce problème en Python. Ce sont des structures qui associent une clé (un nom, une étiquette) à une valeur. Au lieu de demander "donne-moi l'élément numéro 2", tu demandes "donne-moi l'âge de Pierre".

A retenir

  • Un dictionnaire stocke des paires clé → valeur.
  • Les clés sont uniques (pas de doublon possible).
  • L'accès à une valeur se fait par sa clé, pas par un index.
  • On peut ajouter, modifier et supprimer des entrées à tout moment.
  • On peut parcourir un dictionnaire sur ses clés, ses valeurs, ou les deux à la fois.

Analogie fil rouge

Imagine un carnet d'adresses papier : chaque page a un nom (la clé) et, en face, un numéro de téléphone (la valeur). Pour retrouver le numéro de Marie, tu cherches directement "Marie" — tu n'as pas besoin de compter les pages. C'est exactement ce que fait un dictionnaire Python.

1 — Pourquoi un dictionnaire plutôt qu'une liste ?

Le problème avec deux listes parallèles

Supposons que tu veuilles stocker les âges de quatre étudiants. Avec des listes, ça ressemble à ceci.

Python
students = ["Peter", "Mary", "George", "Emma"]  # liste des prénoms
student_ages = [24, 25, 22, 20]                  # liste des âges, dans le même ordre

# Pour afficher chaque étudiant avec son âge,
# on doit utiliser enumerate() pour avoir l'index
for index, student in enumerate(students):       # index = 0, 1, 2, 3 tour à tour
    age = student_ages[index]                    # on récupère l'âge via l'index
    print(f"- {student} ({age} years old)")      # on affiche le résultat

Ca fonctionne, mais c'est fragile : si tu changes l'ordre d'une liste sans changer l'autre, tout est faux. Et pour trouver l'âge de "Peter", tu ne peux pas écrire student_ages["Peter"] — ça ne marche pas avec une liste.

La solution : un dictionnaire

Python
# On regroupe tout dans un seul dict :
# chaque prénom (clé) est directement lié à son âge (valeur)
students_age = {
    "Peter": 24,    # clé "Peter"  →  valeur 24
    "Mary": 25,     # clé "Mary"   →  valeur 25
    "George": 22,   # clé "George" →  valeur 22
    "Emma": 20,     # clé "Emma"   →  valeur 20
}

# Accès direct, clair et sans risque d'erreur d'index
print(students_age["Peter"])   # affiche 24
💡

Un dictionnaire rend le code plus lisible et plus robuste. Quand les données ont un sens de type "nom → valeur", le dictionnaire est presque toujours le bon choix.

2 — Créer un dictionnaire

Un dictionnaire se crée avec des accolades {}. Chaque entrée est une paire clé: valeur, séparée des autres par une virgule.

Python
# Dictionnaire vide — on le remplira plus tard
empty_dict = {}

# Dictionnaire avec des valeurs initiales
london = {
    "country": "England",          # clé "country"    →  valeur "England"
    "population": 8_982_000,       # clé "population"  →  valeur 8982000
}                                  # le _ dans les nombres est juste pour la lisibilité
🧠

Les clés sont le plus souvent des chaînes de caractères (strings), mais elles peuvent aussi être des nombres ou des tuples. En revanche, elles ne peuvent pas être des listes (on verra pourquoi dans la section sur les erreurs courantes).

3 — Lire des valeurs

Avec les crochets []

C'est la façon la plus directe. On écrit le nom du dictionnaire, puis la clé entre crochets.

Python
london["country"]      # renvoie "England"
london["population"]   # renvoie 8982000
⚠️

Si la clé n'existe pas, Python lève une erreur KeyError et le programme s'arrête. Exemple : london["mayor"] → KeyError: 'mayor'. Pour éviter ça, utilise .get() (voir juste en dessous).

Avec .get() — la lecture sécurisée

La méthode .get() ne plante jamais : si la clé est absente, elle renvoie None (ou une valeur par défaut que tu choisis).

Python
london.get("country")                         # renvoie "England" — clé présente
london.get("mayor")                           # renvoie None      — clé absente, pas d'erreur
london.get("mayor", "information inconnue")   # renvoie "information inconnue" — valeur par défaut
✅

En pratique, préfère toujours .get() quand tu n'es pas certain que la clé existe. Ca évite les crashs inattendus.

4 — Ajouter et modifier des entrées

Bonne nouvelle : la syntaxe est exactement la même pour ajouter une nouvelle clé et pour modifier une clé existante.

Python
# Ajouter une nouvelle clé qui n'existait pas encore
london["star_monument"] = "Big Ben"    # le dict contient maintenant 3 entrées

# Modifier la valeur d'une clé déjà présente
london["population"] = 8_982_001       # la valeur précédente est remplacée

Pour mettre à jour plusieurs clés en une seule fois, utilise .update().

Python
# .update() accepte un dict en argument et fusionne les valeurs
london.update({"population": 9_000_000, "country": "England"})
# Les clés existantes sont mises à jour, les nouvelles sont ajoutées
⚠️

Les clés d'un dictionnaire sont uniques. Si tu assigne une valeur à une clé déjà existante, l'ancienne valeur est silencieusement écrasée sans avertissement. Fais attention lors de la construction d'un dict avec des doublons.

5 — Supprimer une entrée

Deux façons de supprimer une entrée.

Python
# del supprime la clé et sa valeur, ne retourne rien
del london["star_monument"]    # "star_monument" disparaît du dict

# .pop() supprime ET retourne la valeur supprimée (utile si tu en as besoin)
population_retiree = london.pop("population")    # renvoie 9_000_000
print(population_retiree)                        # affiche 9000000
💡

Utilise del quand tu veux juste supprimer. Utilise .pop() quand tu veux récupérer la valeur au passage.

6 — Méthodes utiles

Connaître la taille d'un dictionnaire

len() renvoie le nombre de clés dans le dictionnaire.

Python
london = {"country": "England", "population": 8_982_000}
len(london)    # renvoie 2 — il y a 2 paires clé/valeur

Vérifier si une clé existe

L'opérateur in renvoie True ou False.

Python
"country" in london      # True  — la clé existe
"mayor" in london        # False — la clé n'existe pas
🧠

Combiner in et .get() couvre tous tes besoins de lecture sécurisée. in est utile quand tu veux tester l'existence avant d'agir ; .get() est utile quand tu veux lire sans risquer une erreur.

7 — Parcourir un dictionnaire (boucles)

Un dictionnaire peut être parcouru de trois manières différentes selon ce dont tu as besoin.

7.1 Parcourir les clés

Par défaut, une boucle for sur un dictionnaire donne les clés.

Python
london = {"country": "England", "population": 8_982_000}

for key in london:          # key vaut "country", puis "population"
    print(key)              # affiche chaque clé
# Résultat :
# country
# population

7.2 Parcourir les valeurs

La méthode .values() donne accès aux valeurs uniquement.

Python
for value in london.values():    # value vaut "England", puis 8982000
    print(value)                 # affiche chaque valeur
# Résultat :
# England
# 8982000

7.3 Parcourir les paires clé/valeur (le plus courant)

La méthode .items() donne des paires (clé, valeur). On les "déstructure" directement dans la boucle avec deux variables.

Python
for key, value in london.items():              # chaque tour donne une paire
    print(f"{key} -> {value}")                 # on affiche les deux ensemble
# Résultat :
# country -> England
# population -> 8982000
✅

.items() est la façon la plus lisible et la plus utilisée pour parcourir un dictionnaire. Mémorise-la en priorité.

8 — Dict comprehension (bonus)

La dict comprehension permet de créer un dictionnaire en une seule ligne, à partir d'une logique. C'est l'équivalent des list comprehensions, mais pour les dicts.

Construire un dict depuis deux listes

Python
students = ["Peter", "Mary", "George", "Emma"]    # liste des prénoms
student_ages = [24, 25, 22, 20]                   # liste des âges correspondants

# zip() associe les deux listes élément par élément : ("Peter", 24), ("Mary", 25)...
# la comprehension transforme chaque paire en entrée clé/valeur
student_ages_dict = {key: value for key, value in zip(students, student_ages)}

print(student_ages_dict)
# Résultat : {'Peter': 24, 'Mary': 25, 'George': 22, 'Emma': 20}

Transformer les valeurs d'un dict existant

Python
scores = {"Alice": 10, "Bob": 8, "Clara": 15}    # scores d'origine

# On crée un nouveau dict en doublant chaque score
scores_doubles = {k: v * 2 for k, v in scores.items()}   # k = clé, v = valeur

print(scores_doubles)
# Résultat : {'Alice': 20, 'Bob': 16, 'Clara': 30}
🧠

La dict comprehension suit toujours le même schéma : {nouvelle_clé: nouvelle_valeur for clé, valeur in source}. Elle est très appréciée car concise, mais ne sacrifie jamais la lisibilité — si la logique est complexe, une boucle classique reste préférable.

9 — Erreurs courantes à éviter

⚠️

Accéder à une clé inexistante avec []

Python
london = {"country": "England"}
london["population"]      # KeyError — la clé "population" n'existe pas
london.get("population")  # None     — pas d'erreur, retourne None
⚠️

Utiliser une liste comme clé

Les clés doivent être des valeurs "hashables" (immuables). Les listes ne le sont pas.

Python
d = {[1, 2]: "valeur"}   # TypeError: unhashable type: 'list'
d = {(1, 2): "valeur"}   # OK — un tuple est immuable, donc utilisable comme clé
⚠️

Oublier que les clés sont uniques

Si tu définis deux fois la même clé dans un dict, la deuxième valeur écrase la première, sans avertissement.

Python
d = {"age": 20, "age": 30}   # Python accepte ça sans erreur
print(d)                      # {"age": 30} — la valeur 20 a disparu
⚠️

Modifier un dict pendant qu'on itère dessus

Supprimer ou ajouter des clés en pleine boucle provoque une RuntimeError.

Python
for key in london:
    del london[key]           # RuntimeError: dictionary changed size during iteration

# Solution : itérer sur une copie des clés
for key in list(london.keys()):
    del london[key]           # OK — on itère sur une liste figée

📌 TL;DR

  • Un dictionnaire (dict) stocke des paires clé: valeur entre accolades {}.
  • On accède à une valeur avec d["cle"] (plante si la clé est absente) ou d.get("cle") (retourne None, plus sûr).
  • On ajoute ou modifie une entrée avec d["cle"] = valeur — même syntaxe dans les deux cas.
  • On supprime avec del d["cle"] (sans retour) ou d.pop("cle") (retourne la valeur supprimée).
  • len(d) donne le nombre de clés ; "cle" in d teste l'existence d'une clé.
  • Pour parcourir : for k in d (clés), for v in d.values() (valeurs), for k, v in d.items() (les deux).
  • Les clés sont uniques — assigner deux fois la même clé écrase silencieusement la première valeur.
  • Les clés doivent être immuables (strings, nombres, tuples) — les listes ne peuvent pas être des clés.
  • La dict comprehension {k: v for k, v in source} crée un dict en une ligne à partir d'une séquence.
  • Ne jamais modifier un dict pendant qu'on itère dessus — itérer sur list(d.keys()) à la place.

A. Concepts du cours

Le dict, hashmap omniprésent

Le dictionnaire Python est une table de hachage : chaque clé est convertie en un entier (le hash), qui sert d'index dans un tableau interne. Conséquence : dict[key] et key in dict sont des opérations en $O(1)$ amorti, quel que soit le nombre de clés.

C'est la structure de données la plus utilisée en Python — elle est partout : namespaces (globals(), locals()), attributs d'objets (obj.__dict__), modules, JSON, configurations.

Python
user = {"name": "Alice", "age": 30, "city": "Paris"}

user["name"]                # "Alice" — accès O(1)
user["age"] = 31            # Modification
user["email"] = "a@x.com"   # Insertion (clé absente)
del user["city"]            # Suppression
"name" in user              # True — test d'appartenance O(1)
len(user)                   # 3
💡

Analogie : un dict est une bibliothèque où chaque livre a un emplacement déterminé par un calcul sur son titre. Quand vous cherchez "Le Petit Prince", la bibliothécaire applique la fonction de hachage au titre, va directement à l'étagère 47, case 12, et trouve le livre instantanément. Pas besoin de parcourir tous les rayons.

Création, lecture, écriture

Trois syntaxes de création :

Python
# 1. Littéral (le plus courant)
d = {"a": 1, "b": 2, "c": 3}

# 2. Constructeur
d = dict(a=1, b=2, c=3)              # Pour des clés qui sont des identifiants
d = dict([("a", 1), ("b", 2)])       # Depuis une liste de paires

# 3. Comprehension
d = {k: v**2 for k, v in [("a", 1), ("b", 2)]}
# {'a': 1, 'b': 4}

L'accès d[key] lève KeyError si la clé n'existe pas. Trois alternatives plus sûres :

Python
d.get("z")              # None si absent (pas d'erreur)
d.get("z", 0)           # 0 par défaut si absent
d.setdefault("z", 0)    # Insère z=0 si absent, renvoie la valeur

setdefault est subtil : il modifie le dict si la clé manque. Utile pour grouper :

Python
words = ["apple", "ant", "banana", "berry"]
groups = {}
for word in words:
    # Ajoute à la liste existante OU crée une nouvelle liste
    groups.setdefault(word[0], []).append(word)
# {'a': ['apple', 'ant'], 'b': ['banana', 'berry']}

Itération sur un dict

Trois méthodes selon ce qu'on veut :

Python
d = {"a": 1, "b": 2, "c": 3}

# Clés seules (par défaut)
for key in d:
    print(key)              # a, b, c

# Clés (plus explicite, identique)
for key in d.keys():
    print(key)

# Valeurs
for value in d.values():
    print(value)            # 1, 2, 3

# Paires (clé, valeur)
for key, value in d.items():
    print(f"{key} = {value}")
👉

Bonne pratique : utilisez toujours .items() quand vous avez besoin des deux. Évitez for k in d: v = d[k] qui fait deux lookups au lieu d'un.

Patterns courants

Comptage d'occurrences :

Python
words = ["apple", "banana", "apple", "cherry", "banana", "apple"]
counts = {}
for word in words:
    counts[word] = counts.get(word, 0) + 1
# {'apple': 3, 'banana': 2, 'cherry': 1}

# Version pythonique
from collections import Counter
counts = Counter(words)
counts.most_common(2)        # [('apple', 3), ('banana', 2)]

Inversion clé/valeur :

Python
d = {"a": 1, "b": 2, "c": 3}
inverted = {v: k for k, v in d.items()}
# {1: 'a', 2: 'b', 3: 'c'}
⚠️

Piège : si plusieurs clés ont la même valeur, l'inversion en perd : {"a": 1, "b": 1} devient {1: "b"} (la dernière écrase). Pour conserver toutes les clés : inverted = {}; for k, v in d.items(): inverted.setdefault(v, []).append(k).

Fusion de deux dicts (Python 3.9+) :

Python
defaults = {"theme": "dark", "lang": "fr"}
user_prefs = {"lang": "en", "font_size": 14}
merged = defaults | user_prefs
# {'theme': 'dark', 'lang': 'en', 'font_size': 14}
# L'opérande de droite gagne en cas de conflit

Filtrage :

Python
scores = {"Alice": 85, "Bob": 60, "Charlie": 92}
passing = {name: score for name, score in scores.items() if score >= 80}
# {'Alice': 85, 'Charlie': 92}

B. Compléments avancés

Comment fonctionne le hashing

Quand vous faites d["alice"] = 42, Python :

  1. Calcule hash("alice") — un entier (généralement très grand).
  2. Prend hash("alice") % capacity pour obtenir un index dans le tableau interne.
  3. Stocke la paire (clé, valeur) à cet index.

Quand deux clés ont la même position (collision), Python utilise un mécanisme appelé open addressing : il cherche le slot suivant disponible. Quand le tableau dépasse 2/3 de remplissage, il est reconstruit avec une capacité plus grande pour maintenir l'opération moyenne en $O(1)$.

Conséquence cruciale : les clés doivent être hashables (hash(key) doit être implémenté et stable). En pratique :

TypeHashable ?
int, float, str, bool, NoneOui
tupleOui (si tous ses éléments le sont)
frozensetOui
list, dict, setNon
Classes customOui par défaut (basé sur l'identité)
Python
d = {(1, 2): "ok"}          # OK : tuple hashable
d[[1, 2]] = "fail"          # TypeError: unhashable type: 'list'
⚠️

Question d'entretien classique : "Pourquoi une liste ne peut-elle pas être une clé de dict ?" Réponse : parce qu'elle est mutable. Si vous l'utilisiez comme clé puis la modifiiez, son hash changerait — et le dict ne saurait plus retrouver la valeur associée. Python interdit donc cet usage à la racine. Le tuple est safe parce qu'il est immuable par conception.

defaultdict et Counter

collections propose deux variantes du dict qui éliminent du boilerplate :

defaultdict : crée automatiquement la valeur par défaut quand on accède à une clé inexistante.

Python
from collections import defaultdict

# Sans defaultdict
groups = {}
for word in words:
    if word[0] not in groups:
        groups[word[0]] = []
    groups[word[0]].append(word)

# Avec defaultdict — bien plus concis
groups = defaultdict(list)        # La "factory" est list, qui crée []
for word in words:
    groups[word[0]].append(word)  # Si la clé n'existe pas, crée [] automatiquement

L'argument de defaultdict est une factory : une fonction sans argument qui renvoie la valeur par défaut.

  • defaultdict(list) → []
  • defaultdict(int) → 0 (utile pour des compteurs)
  • defaultdict(dict) → {} (nested dicts)

Counter : compteur multi-éléments avec API riche.

Python
from collections import Counter

c = Counter("mississippi")
# Counter({'i': 4, 's': 4, 'p': 2, 'm': 1})

c.most_common(2)            # [('i', 4), ('s', 4)]
c["i"]                      # 4
c["z"]                      # 0 — pas de KeyError pour les clés absentes
c.update("missouri")        # Incrémente les compteurs

# Opérations arithmétiques entre Counters
c1 = Counter("apple")
c2 = Counter("ape")
c1 - c2                     # Counter({'p': 1, 'l': 1}) — soustraction
c1 + c2                     # Counter({'p': 3, 'a': 2, 'e': 2, 'l': 1})

Dict ordonnés et dict comprehensions

Depuis Python 3.7, les dicts conservent garantissement l'ordre d'insertion. Avant 3.6, l'ordre était indéterminé.

Cette garantie permet des patterns intéressants :

Python
# Dédupliquer en préservant l'ordre
items = [3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5]
unique = list(dict.fromkeys(items))
# [3, 1, 4, 5, 9, 2, 6]

dict.fromkeys(items) crée un dict avec les éléments comme clés et None comme valeur. Les clés étant uniques par définition, on dédoublonne. L'ordre d'insertion est préservé.

Dict comprehensions suivent la même syntaxe que les list comprehensions :

Python
# Carré de chaque entier
squares = {x: x**2 for x in range(5)}
# {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}

# Inverser un dict
inverted = {v: k for k, v in d.items()}

# Conditionner
filtered = {k: v for k, v in d.items() if v > 0}

C. Questions de compréhension

Q1 : Quelle est la complexité moyenne et pire-cas de key in dict, et qu'est-ce qui peut faire dégénérer le dict en O(n) ?

💡 Voir l'indice

Collisions de hash.

✅ Voir la réponse

En moyenne : $O(1)$.

Pire cas théorique : $O(n)$, qui se produit quand toutes les clés ont le même hash — donc toutes finissent à la même position et Python doit parcourir une chaîne linéaire.

En pratique cela n'arrive que si quelqu'un crée des clés conçues exprès pour entrer en collision — c'est l'attaque hash flooding, qui a déjà été un vecteur d'attaque DoS sur des serveurs web.

Python a depuis introduit une randomisation du hash de strings au démarrage de l'interpréteur (PYTHONHASHSEED) pour rendre cette attaque imprédictible. Pour les dicts construits par votre application avec des clés normales (strings, ints, tuples), vous restez en $O(1)$ amorti.

Q2 : Vous voulez stocker en dict des objets de classe Point (avec .x et .y). Pourquoi ça ne marche pas par défaut, et comment réparer ?

💡 Voir l'indice

__hash__ et __eq__.

✅ Voir la réponse

Par défaut, les classes Python ont un __hash__ basé sur l'identité (id en mémoire), donc deux objets Point(1, 2) et Point(1, 2) créés séparément ont des hash différents et sont considérés comme des clés différentes.

Python
class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y

p1 = Point(1, 2)
p2 = Point(1, 2)
d = {p1: "a"}
d[p2]              # KeyError ! Alors qu'on a les mêmes valeurs

Pour qu'ils soient interchangeables, il faut implémenter __hash__ ET __eq__ de manière cohérente : deux objets égaux doivent avoir le même hash.

Solution moderne avec @dataclass(frozen=True) :

Python
from dataclasses import dataclass

@dataclass(frozen=True)   # frozen=True génère __hash__ ET __eq__ automatiquement
class Point:
    x: int
    y: int

p1 = Point(1, 2)
p2 = Point(1, 2)
d = {p1: "a"}
d[p2]              # 'a' — fonctionne !

Le frozen=True fait deux choses : (1) rend l'instance immuable (le hash reste stable) et (2) génère __hash__ basé sur les valeurs des champs.

Q3 : Vous avez un dict de dict de listes (ex: data["users"]["alice"]["scores"]) et vous voulez data["users"]["bob"]["scores"].append(95). Que se passe-t-il si "bob" n'existe pas, et comment gérer proprement ?

💡 Voir l'indice

Nested defaultdict.

✅ Voir la réponse

Vous obtenez KeyError: 'bob' parce que data["users"]["bob"] n'existe pas.

Trois solutions de plus en plus élégantes :

1. setdefault chaîné — verbeux mais explicite :

Python
data["users"].setdefault("bob", {}).setdefault("scores", []).append(95)

2. defaultdict à deux niveaux — beaucoup plus propre :

Python
from collections import defaultdict
data = {"users": defaultdict(lambda: defaultdict(list))}
data["users"]["bob"]["scores"].append(95)    # Crée tout automatiquement

3. Arbre récursif pour des structures profondes à profondeur variable :

Python
from collections import defaultdict

def tree():
    return defaultdict(tree)

data = tree()
data["users"]["bob"]["scores"] = [95]   # Crée toute la chaîne automatiquement
data["users"]["alice"]["info"]["city"] = "Paris"  # Profondeur arbitraire

Tip pratique : pour de la donnée structurée plus complexe, ne réinventez pas la roue — utilisez pydantic ou des dataclass qui apportent validation et auto-complétion IDE.
