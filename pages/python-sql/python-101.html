def full_name(first_name, last_name):

🧭 Introduction

Python est un langage de programmation : une manière d'écrire des instructions que l'ordinateur comprend. Avant d'écrire des programmes utiles, il faut apprendre les briques de base : comment stocker une information (une variable), quels genres d'informations existent (un texte ? un nombre ?), comment prendre une décision (si… alors…) et comment créer une petite machine réutilisable (une fonction).

Ce cours est une visite guidée de ces briques. À la fin, tu sauras lire et écrire du Python très simple — le genre qu'on utilise dans tous les scripts et projets data, sans exception.

À retenir absolument

  • En Python, chaque valeur a un type (texte, entier, booléen…) et ce type détermine ce que tu peux en faire.
  • Une variable n'est qu'une étiquette collée sur une valeur pour la retrouver plus tard.
  • Une fonction est une recette qui transforme des ingrédients (les paramètres) en un résultat.
  • Les conditions (if / else) permettent au programme de faire des choix différents selon la situation.
  • L'indentation (les espaces au début des lignes) n'est pas décorative en Python : elle définit la structure du code.

Analogie fil rouge. Imagine Python comme une cuisine. Les types sont les ingrédients (farine, œufs, lait). Les variables sont les bocaux étiquetés où tu ranges les ingrédients. Les fonctions sont tes recettes. Les conditions sont tes choix : « si j'ai du chocolat, je fais un gâteau au chocolat, sinon je fais un gâteau nature ». Garde cette image en tête tout le cours.

Dans ce cours reformulé, nous allons d'abord découvrir les types de données, puis manipuler du texte (strings) et des nombres, ensuite comprendre la différence fonction / méthode, stocker des valeurs dans des variables, écrire nos propres fonctions, et enfin prendre des décisions avec les conditions.

1) Les types de données : les ingrédients de Python

Chaque valeur que tu manipules en Python appartient à un type. Le type, c'est la « nature » de l'ingrédient : est-ce du texte, un nombre entier, un nombre à virgule, une liste de choses ?

Voici les types de base que tu vas croiser tout le temps.

Ici, on écrit une série de valeurs et, à droite de chaque ligne, un commentaire qui indique son type :

Python
"Hello World!"               # str   — une chaîne de caractères (du texte)
12                           # int   — un nombre entier
3.14                         # float — un nombre à virgule (flottant)
["hi", 12, 3.14]             # list  — une liste ordonnée de plusieurs valeurs
{"name": "Ben", "age": 31}   # dict  — un dictionnaire : des paires clé → valeur
(250, 4)                     # tuple — comme une liste, mais non modifiable
True                         # bool  — un booléen : vrai
False                        # bool  — un booléen : faux

Récap : tu viens de rencontrer les 7 types de base. Dans la vraie vie, tu passeras 90 % de ton temps avec str, int, float, list, dict et bool.

Pour demander à Python le type d'une valeur, on utilise la fonction type() :

Python
type(12)   # Python répond : <class 'int'>, autrement dit "c'est un entier"
💡

Astuce : quand tu as un doute sur ce qu'est une variable ou pourquoi ton code plante, glisse un print(type(ma_variable)) pour vérifier. 90 % des bugs de débutant viennent d'un type inattendu (un « 42 » texte au lieu d'un 42 entier, par exemple).

2) Les strings : manipuler du texte

Une string (chaîne de caractères) est simplement du texte entre guillemets. Python accepte les guillemets simples '...' ou doubles "..." — au choix, tant que tu restes cohérent.

Python
"hello" == 'hello'                       # True — les deux guillemets produisent la même string
"hello world!".upper()                   # "HELLO WORLD!" — la méthode .upper() met tout en majuscules
" scraped_email@email.com   ".strip()    # "scraped_email@email.com" — .strip() enlève les espaces autour

Résumé : .upper(), .lower(), .strip(), .capitalize() sont tes amis pour nettoyer du texte venu de l'extérieur (un fichier, un formulaire web…).

Concaténer (coller) des strings

Pour assembler deux morceaux de texte, on les additionne avec + :

Python
"hello" + " " + "world!"   # "hello world!" — trois strings collées l'une à l'autre

f-strings : interpoler des valeurs dans un texte

Une f-string est un texte qui contient des « trous » qu'on remplit avec des valeurs. Elle commence par la lettre f juste avant le guillemet, et les trous sont écrits entre accolades {} :

Python
age = 31                          # on stocke l'âge dans une variable
f"J'ai {age} ans"                 # "J'ai 31 ans" — {age} est remplacé par la valeur de age
f"two: {1 + 1}"                   # "two: 2" — on peut même mettre un calcul dans les accolades

Explication : Python voit le f devant le guillemet, comprend qu'il doit remplacer les {...} par leur valeur, et produit un texte fini.

⚠️

Piège fréquent : oublier le f. Sans lui, les accolades sont prises au pied de la lettre et affichées telles quelles.

Python
"two: {1 + 1}"    # affiche littéralement "two: {1 + 1}" — pas de calcul, pas de remplacement
f"two: {1 + 1}"   # "two: 2" — correct

Convertir du texte en nombre et vice-versa

Quand tu lis un nombre depuis un fichier ou un formulaire, il arrive souvent sous forme de texte. Pour faire des maths, il faut le convertir :

Python
int("31")       # 31    — transforme le texte "31" en entier
float("31.2")   # 31.2  — transforme le texte "31.2" en flottant
str(42)         # "42"  — dans l'autre sens : transforme l'entier 42 en texte

Connaître la longueur

Python
len("hello")   # 5 — len() compte le nombre de caractères de la string
✅

Récap express : une string se nettoie avec .strip(), se met en majuscules avec .upper(), se combine avec +, s'interpole avec f"...", et se compte avec len().

3) Fonction vs méthode : deux manières d'appeler du code

C'est une distinction qui déroute au début, mais elle est simple.

Une fonction est autonome. Tu l'appelles par son nom, en mettant les arguments entre parenthèses : nom_de_fonction(arguments).

Python
str(42)     # "42"   — str() est une fonction : on lui donne 42, elle renvoie le texte "42"
str(42.5)   # "42.5"
str(True)   # "True"

Une méthode est attachée à un objet. Tu l'appelles via un point : objet.nom_de_methode(arguments). La méthode agit sur l'objet qui est devant le point.

Python
"Python".startswith("P")   # True — .startswith() est une méthode de l'objet string "Python"

Et quand tu appelles une méthode sur un objet qui ne la possède pas, Python râle :

Python
(123).startswith(1)        # AttributeError : les entiers n'ont pas de méthode .startswith()

Analogie : une fonction est un outil posé sur l'établi, tu le prends pour agir sur quelque chose. Une méthode est un outil intégré à un objet, comme l'ouverture automatique d'une voiture — c'est la voiture qui sait ouvrir ses propres portes.

4) Les nombres

Opérations de base

Python
1 + 2    # 3  — addition
2 * 4    # 8  — multiplication
3 % 2    # 1  — modulo : le reste de la division entière de 3 par 2
2 ** 3   # 8  — puissance : 2 à la puissance 3, soit 2×2×2

Le modulo (%) est très utile : il permet de savoir si un nombre est pair (n % 2 == 0), multiple de 10, etc.

Conversion en texte

Python
str(12)     # "12"
str(12.4)   # "12.4"

Arrondir

Python
round(12.4)   # 12 — round() arrondit au plus proche
🧠

Pour aller plus loin : round(12.5) renvoie… 12 en Python, pas 13 ! Python utilise l'arrondi banquier : en cas d'égalité, il arrondit vers le nombre pair le plus proche. Bon à savoir pour ne pas être surpris.

5) Les variables : ranger une valeur dans un bocal

Une variable, c'est juste une étiquette que tu colles sur une valeur pour pouvoir la réutiliser. L'opérateur = sert à faire ce collage.

Ici, on déclare la variable age, on l'affiche, puis on la modifie :

Python
age = 17                                 # on range 17 dans le bocal "age"
print(f"You are {age} years old")        # "You are 17 years old"

print("Lucky you, it's your birthday!")
age = age + 1                            # on lit la valeur actuelle de age (17), on ajoute 1, et on range le résultat (18) dans age
print(f"You are now {age}")              # "You are now 18"

Explication ligne par ligne : age = age + 1 se lit de droite à gauche. Python calcule d'abord age + 1 (soit 17 + 1 = 18), puis range le résultat dans age. À la fin, age vaut 18.

⚠️

Piège fréquent : confondre = (affectation : « je range ») et == (comparaison : « est-ce que c'est égal ? »).

Python
age = 18    # affectation : on range 18 dans age
age == 18   # comparaison : renvoie True si age vaut 18

6) Écrire ses propres fonctions

Pourquoi ?

Pour ne pas réécrire le même code plusieurs fois. Si tu dois saluer 10 personnes, tu n'écris pas 10 fois la phrase : tu crées une fonction say_hi(name) que tu appelles 10 fois avec un nom différent.

Exemple simple

On définit la fonction avec def, on donne un nom (say_hi), puis entre parenthèses on liste les paramètres (les ingrédients qu'elle attend). Le corps de la fonction est indenté (décalé vers la droite). return indique ce que la fonction renvoie.

Python
def say_hi(name):              # on définit la fonction say_hi qui attend un paramètre "name"
    return f"Hi {name}."       # elle renvoie le texte "Hi " suivi du nom passé en argument

print(say_hi("Alex"))          # affiche "Hi Alex."
print(say_hi("Edward"))        # affiche "Hi Edward."

Résumé : tu as défini une recette une seule fois, et tu l'as utilisée deux fois avec des ingrédients différents.

Exemple avec 2 paramètres

Les fonctions peuvent prendre plusieurs paramètres, séparés par des virgules.

Python
def full_name(first_name, last_name):                          # deux paramètres
    name = first_name.capitalize() + " " + last_name.capitalize()  # on met chaque prénom en Majuscule et on les colle
    return name                                                # on renvoie le nom complet

print(full_name("boris", "paillard"))   # "Boris Paillard"
print(full_name("seb", "saunier"))      # "Seb Saunier"
💡

Astuce : vocabulaire à ne jamais confondre.

- Paramètres = les noms écrits dans la définition (first_name, last_name). Ce sont les « cases » que la fonction attend.

- Arguments = les valeurs réellement passées à l'appel ("boris", "paillard"). Ce sont ce qu'on verse dans les cases.

7) Les conditions : faire des choix

if : « si telle condition est vraie, fais ceci »

Python
if age >= 18:
    print("Tu peux voter.")   # cette ligne n'est exécutée que si age est supérieur ou égal à 18

Note l'indentation : tout ce qui est « à l'intérieur » du if doit être décalé de quatre espaces (ou d'une tabulation).

if / else : « sinon, fais autre chose »

Python
if age >= 18:
    print("Tu peux voter.")    # si la condition est vraie
else:
    print("Pas encore.")       # si la condition est fausse

if / elif / else : plusieurs cas

elif est la contraction de « else if » : « sinon, si telle autre condition ».

Python
if note >= 16:
    print("Très bien")
elif note >= 12:
    print("Bien")              # testé seulement si la première condition a échoué
elif note >= 10:
    print("Passable")
else:
    print("Insuffisant")       # si aucune condition précédente n'a été vraie

Python teste les conditions dans l'ordre et s'arrête à la première qui est vraie.

L'opérateur ternaire : une condition en une ligne

Quand ta condition est très simple, tu peux l'écrire en une seule ligne :

Python
age = 18
print("You can vote" if age >= 18 else "You can't vote")   # affiche "You can vote"

Lecture : « affiche X si condition, sinon Y ». L'ordre est un peu spécial (le résultat d'abord, puis la condition), mais on s'y habitue vite.

8) Combiner des conditions : and, or, not

Python
True  and True    # True  — les deux doivent être vrais
True  and False   # False — l'un est faux, donc tout est faux
False and False   # False

True  or False    # True  — il suffit qu'un seul soit vrai
False or False    # False — les deux sont faux

Exemple concret : pour qu'un utilisateur puisse se connecter, il faut un login correct et un mot de passe correct. login_ok and password_ok.

9) La valeur spéciale None et les valeurs « vides »

En Python, presque toutes les valeurs sont considérées comme vraies dans une condition. Les exceptions principales — que Python considère comme fausses — sont :

  • None (l'absence de valeur)
  • False
  • les « conteneurs vides » : liste vide [], dictionnaire vide {}, string vide "", et le nombre 0

On peut donc tester directement une variable dans un if, sans comparer explicitement :

Python
name = "Jenny"

if name:                           # équivaut à "si name n'est ni vide ni None"
    print(f"Hi {name}")            # "Hi Jenny"
else:
    print("Name not provided")

Résumé : if name: est une manière courte et idiomatique de dire « si on a bien un nom ».

⚠️

Piège fréquent : oublier les parenthèses sur un appel de méthode.

Python
"hello".upper    # ❌ renvoie la méthode elle-même (un objet), pas le résultat
"hello".upper()  # ✅ renvoie "HELLO"
⚠️

Piège fréquent : indentation incorrecte. Chaque bloc (fonction, if, boucle) doit être décalé de manière cohérente.

Python
def say_hi(name):
return f"Hi {name}"     # ❌ IndentationError — le return n'est pas indenté

def say_hi(name):
    return f"Hi {name}" # ✅

       print("Anonyme")

📌 TL;DR

  • Python distingue plusieurs types : str (texte), int (entier), float (virgule), list, dict, bool. Vérifie avec type().
  • Les strings se nettoient avec .strip(), se concatènent avec + et s'interpolent avec les f-strings f"Bonjour {name}".
  • Une fonction est autonome (str(42)), une méthode est attachée à un objet ("hi".upper()).
  • Une variable stocke une valeur (age = 17). = range, == compare.
  • On crée ses fonctions avec def nom(params): et on renvoie un résultat avec return.
  • Les conditions if / elif / else permettent les choix. L'indentation structure le code.
  • and, or, not combinent les conditions. None, 0, "", [] sont considérés comme faux.
  • 90 % des erreurs de débutant : oubli du f, confusion = vs ==, indentation, oubli des parenthèses sur une méthode.

A. Concepts du cours

Variables, types et duck typing

Python est un langage dynamiquement typé : une variable est juste une étiquette qui peut pointer vers n'importe quel objet, et changer de type au cours de l'exécution. Contrairement à C ou Java, vous ne déclarez jamais le type. Le type est attaché à l'objet, pas à la variable.

Python
x = 42         # x pointe vers un int
x = "hello"    # x pointe maintenant vers un str — c'est légal
x = [1, 2, 3]  # x pointe maintenant vers une list
# L'int 42 existe toujours en mémoire, mais plus aucune variable ne le référence

Types built-in essentiels :

TypeExempleMutable ?Hashable ?
int42, -3, 10**100NonOui (précision arbitraire !)
float3.14, 1e-5NonOui
boolTrue, FalseNonOui
str"hello", 'world'NonOui
list[1, 2, 3]OuiNon
tuple(1, 2, 3)NonOui (si éléments hashables)
dict{"a": 1}OuiNon
set{1, 2, 3}OuiNon
NoneNone(singleton unique)Oui

Le duck typing est la philosophie de Python : "If it walks like a duck and quacks like a duck, it's a duck". Au lieu de vérifier le type d'un objet, on vérifie qu'il se comporte comme attendu.

Python
def total(items):
    return sum(items)

total([1, 2, 3])              # 6 (list)
total((1, 2, 3))              # 6 (tuple)
total({1, 2, 3})              # 6 (set)
total(range(4))               # 6 (range object)
total(x for x in [1, 2, 3])   # 6 (generator)

sum ne demande pas "es-tu une liste ?" mais "es-tu itérable et tes éléments supportent-ils l'addition ?".

💡

Analogie : duck typing, c'est embaucher un musicien sans demander son diplôme. Vous lui passez une partition, s'il joue juste, c'est bon. Le typage statique de Java/C++, c'est exiger un Conservatoire avant l'audition.

Opérateurs et expressions

Arithmétiques :

Python

Comparaisons :

Python
a == b        # Égalité de valeur
a != b        # Inégalité
a is b        # Identité (même objet) — à réserver à None, True, False
a < b <= c    # Chaînage : équivalent à (a < b) and (b <= c)

Logiques court-circuitées :

Python

0 and "x"            # 0 — premier falsy

Contrôle de flux

Python
# Expression conditionnelle (ternary)
grade = "A" if score >= 80 else "B"

# for (TOUJOURS sur un iterable, pas sur un index)
for i, word in enumerate(words):
    print(i, word)

# while
n = 0
while n < 5:
    print(n); n += 1

Idiome Python : toujours itérer sur l'iterable, jamais sur les indices :

Python
# ❌ Mauvais
for i in range(len(words)):
    print(words[i])

# ✓ Bon
for word in words:
    print(word)

Fonctions

Python
def compute_score(predictions, labels, weight=1.0):
    """Compute weighted accuracy.

    Args:
        predictions: Liste de labels prédits
        labels: Liste de vrais labels
        weight: Multiplicateur appliqué au résultat

    Returns:
        Weighted accuracy (float).
    """
    correct = sum(p == l for p, l in zip(predictions, labels))
    return weight * correct / len(labels)

*args / </strong>kwargs** capturent un nombre arbitraire d'arguments :

Python
def stats(*nums, **opts):
    print(nums)        # Tuple des positionnels
    print(opts)        # Dict des nommés

stats(1, 2, 3, mean=True)
# (1, 2, 3)
# {'mean': True}
👉

Bonne pratique : préférez les arguments nommés dès que vous avez plus de 2-3 paramètres. train(X, y, test_size=0.2, random_state=42) est bien plus lisible que train(X, y, 0.2, 42).

Type hints et mypy

Python
from typing import Optional

def divide(a: float, b: float) -> Optional[float]:
    if b == 0:
        return None
    return a / b

def process(items: list[int]) -> dict[str, int]:
    return {"count": len(items), "sum": sum(items)}

mypy vérifie statiquement avant d'exécuter :

Bash

Python 3.10+ accepte str | int | None au lieu de Union[str, int, None].

💡

Analogie : les type hints, c'est mettre des étiquettes sur les bacs de votre cuisine. Python ne vous oblige pas à les respecter, mais quand vous cherchez quelque chose à 3 h du matin, vous vous remerciez.

Mutabilité et passage d'arguments

Python passe les arguments par référence d'objet :

Python
def add_one(x):
    x += 1            # Crée un nouvel int, n'affecte pas l'original
a = 5; add_one(a); print(a)    # 5 (int immutable)

def append_zero(lst):
    lst.append(0)     # Modifie l'objet pointé
l = [1, 2]; append_zero(l); print(l)    # [1, 2, 0] — la liste a été modifiée !

Piège des arguments par défaut mutables :

Python
def append_to(item, lst=[]):    # ❌ NE FAITES JAMAIS ÇA
    lst.append(item)
    return lst

append_to(1)   # [1]
append_to(2)   # [1, 2] !! la même liste est réutilisée

Fix :

Python
def append_to(item, lst=None):
    if lst is None:
        lst = []          # Nouvelle liste à chaque appel
    lst.append(item)
    return lst

Le GIL et ses conséquences

Le GIL (Global Interpreter Lock) empêche plusieurs threads Python d'exécuter du bytecode en même temps. threading ne donne pas de speedup CPU sur du code Python pur.

Trois solutions selon le profil :

  1. I/O-bound (HTTP, fichier, DB) : threading ou asyncio — le GIL est libéré pendant les attentes.
  2. CPU-bound : multiprocessing lance plusieurs processus Python indépendants.
  3. Calcul numérique : NumPy/Pandas/PyTorch tournent en C et libèrent le GIL.
Python
# I/O-bound
from concurrent.futures import ThreadPoolExecutor
with ThreadPoolExecutor(max_workers=10) as ex:
    results = list(ex.map(requests.get, urls))

# CPU-bound
from concurrent.futures import ProcessPoolExecutor
with ProcessPoolExecutor(max_workers=4) as ex:
    results = list(ex.map(heavy_compute, dataset))
⚠️

Question d'entretien classique : "Pourquoi mon programme avec 8 threads n'est-il pas 8x plus rapide ?" Réponse : le GIL. Contournements : multiprocessing, NumPy, ou Go/Rust.

C. Questions de compréhension

Q1 : Pourquoi 0.1 + 0.2 == 0.3 retourne-t-il False en Python ?

💡 Voir l'indice

Représentation flottante IEEE 754.

✅ Voir la réponse

Parce que 0.1, 0.2 et 0.3 ne sont pas représentables exactement en binaire flottant IEEE 754. 0.1 est en fait stocké comme 0.1000000000000000055.... La somme 0.1 + 0.2 donne 0.30000000000000004, pas égal à la représentation de 0.3.

Règle universelle : ne jamais comparer deux floats avec ==. Utilisez :

Python
import math
math.isclose(0.1 + 0.2, 0.3, rel_tol=1e-9)   # True

# ou NumPy
np.isclose(0.1 + 0.2, 0.3)                    # True

Pour des montants financiers, utilisez decimal.Decimal qui fait du calcul en base 10 :

Python
from decimal import Decimal
Decimal("0.1") + Decimal("0.2") == Decimal("0.3")   # True

C'est une question d'entretien éliminatoire en numérique.

Q2 : True == 1 et True + 1 == 2 retournent True. Pourquoi ?

💡 Voir l'indice

Héritage de bool.

✅ Voir la réponse

Parce que bool est une sous-classe de int en Python. True est littéralement l'entier 1, False est l'entier 0.

Python
isinstance(True, int)     # True
True + 1                  # 2
sum([True, False, True])  # 2 — très utile pour compter

Ce choix permet sum(x > 0 for x in nums) qui compte les éléments positifs — un idiome très courant.

Pour vérifier strictement le type booléen :

Python
type(x) is bool    # True uniquement pour True/False, pas pour 0/1

Q3 : Vous voulez écrire une fonction qui retourne plusieurs valeurs. Quelle est la meilleure pratique ?

💡 Voir l'indice

Tuples, NamedTuple, ou dataclass ?

✅ Voir la réponse

Option 1 — Tuple (idiome standard pour 2 valeurs simples) :

Python
def divmod_custom(a, b):
    return a // b, a % b            # Renvoie un tuple

q, r = divmod_custom(10, 3)         # Unpacking

Option 2 — NamedTuple (3-4 valeurs avec noms) :

Python
from typing import NamedTuple

class Result(NamedTuple):
    score: float
    n_iter: int
    converged: bool

r = train()
print(r.score)    # Accès par nom

Option 3 — @dataclass (résultats riches avec validation/méthodes) :

Python
from dataclasses import dataclass

@dataclass
class TrainingResult:
    score: float
    n_iter: int
    history: list

Règle : plus la valeur de retour est centrale au domaine, plus elle mérite une structure nommée. Tuple pour une utilité ponctuelle ; dataclass pour un concept métier central.
