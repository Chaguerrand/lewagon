1) Supprimer des éléments : pop, remove et del

Ces trois outils suppriment des éléments, mais de manières différentes.

1.1 pop — supprimer par index et récupérer la valeur

Python
fruits = ["pomme", "banane", "cerise"]
x = fruits.pop(1)   # supprime l'index 1
print(x)            # "banane"
print(fruits)       # ["pomme", "cerise"]

fruits.pop()        # sans argument → supprime le dernier élément
  • Retourne l'élément supprimé
  • Fonctionne sur les listes
  • Existe aussi sur les dicts : d.pop("clé")

1.2 remove — supprimer par valeur

Python
fruits = ["pomme", "banane", "cerise"]
fruits.remove("banane")   # cherche et supprime la 1ère occurrence
print(fruits)              # ["pomme", "cerise"]
  • Ne retourne rien (None)
  • Lève une ValueError si la valeur n'existe pas
  • Supprime uniquement la première occurrence

1.3 del — supprimer par index ou slice (mot-clé)

Python
fruits = ["pomme", "banane", "cerise", "datte"]
del fruits[1]       # supprime l'index 1
print(fruits)       # ["pomme", "cerise", "datte"]

del fruits[1:3]     # supprime un slice
print(fruits)       # ["pomme"]

del fruits          # supprime carrément la variable
  • Ne retourne rien
  • Peut supprimer des slices, des variables, des clés de dict
  • C'est un mot-clé Python, pas une méthode

2) Dictionnaires : .get()

2.1 Le problème sans .get()

Python
personne = {"nom": "Alice", "age": 30}

print(personne["ville"])  # KeyError : la clé n'existe pas

2.2 .get() — accès sans risque de KeyError

Python
personne = {"nom": "Alice", "age": 30}

print(personne.get("nom"))    # "Alice"
print(personne.get("ville"))  # None (pas d'erreur)

2.3 Avec une valeur par défaut

Le 2ème argument définit ce que .get() retourne si la clé est absente.

Python
print(personne.get("ville", "inconnue"))   # "inconnue"
print(personne.get("age", 0))               # 30 (la clé existe)

2.4 Cas d'usage typiques

Compter des occurrences

Python
texte = ["chat", "chien", "chat", "oiseau", "chat"]
compteur = {}

for animal in texte:
    compteur[animal] = compteur.get(animal, 0) + 1

print(compteur)  # {"chat": 3, "chien": 1, "oiseau": 1}

Éviter un if/else verbeux

Python
## Sans .get()
if "ville" in personne:
    ville = personne["ville"]
else:
    ville = "inconnue"

## Avec .get() : une seule ligne
ville = personne.get("ville", "inconnue")

🧭 Introduction

Ce cours regroupe un ensemble de petites notions Python très pratiques que l'on utilise constamment. Ce ne sont pas des concepts théoriques abstraits — ce sont des outils concrets, du quotidien du développeur.

Quand tu débutes, tu passes beaucoup de temps à te demander : "Comment je supprime cet élément ?", "Pourquoi Python me crie dessus avec une KeyError ?", "Comment je fais répéter ce morceau de code ?". Ce cours répond à exactement ces questions.

À retenir avant de commencer :

  • pop, remove et del font tous les trois "supprimer", mais de façons très différentes — les confondre génère des bugs.
  • .get() sur un dictionnaire est une bouée de sauvetage : il évite les erreurs quand une clé n'existe pas.
  • while et for sont les deux façons de faire tourner du code en boucle — chacune a son domaine.
  • Les opérateurs and, or, not permettent de combiner des conditions logiques comme en langage naturel.
  • try/except est le filet de sécurité du code : il attrape les erreurs avant qu'elles ne fassent planter ton programme.

🗑️ Supprimer des éléments : pop, remove et del

Le contexte

Tu as une liste d'éléments et tu veux en enlever un. Python te propose trois outils. Ils font tous "supprimer", mais leur fonctionnement est différent. Choisir le mauvais provoque une erreur.

pop — supprimer par position et récupérer la valeur

pop supprime un élément selon son numéro de position (son index), et en plus il te renvoie l'élément supprimé pour que tu puisses l'utiliser.

Explication : on a une liste de trois fruits. On demande à Python de retirer celui à la position 1 (le deuxième, car on commence à compter à 0), et on récupère ce qui a été retiré dans la variable x.

Python
fruits = ["pomme", "banane", "cerise"]  # liste de départ

x = fruits.pop(1)   # supprime l'élément à l'index 1 ("banane")
                    # et stocke "banane" dans x

print(x)            # affiche : banane
print(fruits)       # affiche : ["pomme", "cerise"]

fruits.pop()        # sans argument → supprime le DERNIER élément

pop retire l'élément ET te le donne en retour. Utile quand tu veux récupérer la valeur avant de la supprimer.

💡

pop fonctionne aussi sur les dictionnaires : mon_dict.pop("cle") supprime la clé et retourne sa valeur.

remove — supprimer par valeur

remove cherche dans la liste la valeur que tu lui donnes, et supprime la première fois qu'elle la trouve. Il ne retourne rien.

Explication : au lieu de dire "retire le deuxième", on dit "retire 'banane'". Python cherche dans la liste et supprime la première occurrence qu'il trouve.

Python
fruits = ["pomme", "banane", "cerise"]  # liste de départ

fruits.remove("banane")  # cherche "banane" et supprime la 1ère occurrence
                         # ne retourne rien (None)

print(fruits)            # affiche : ["pomme", "cerise"]

Après le code : remove est simple mais attention — si la valeur n'existe pas dans la liste, Python lève une ValueError. Et s'il y a plusieurs "banane", seule la première est supprimée.

⚠️

Si tu essaies lst.remove("valeur_inexistante"), Python te retourne une erreur ValueError. Vérifie que la valeur existe avant d'appeler remove, ou utilise un if de garde.

del — supprimer par index, slice, ou même une variable entière

del est un mot-clé Python (pas une méthode). Il peut supprimer un élément à un index donné, une tranche entière de liste (un "slice"), ou même une variable complète.

Explication : del est le plus radical des trois. Il peut supprimer plusieurs éléments d'un coup grâce aux slices, ou même effacer complètement une variable.

Python
fruits = ["pomme", "banane", "cerise", "datte"]  # liste de départ

del fruits[1]       # supprime l'élément à l'index 1 ("banane")
print(fruits)       # affiche : ["pomme", "cerise", "datte"]

del fruits[1:3]     # supprime les éléments aux index 1 et 2 (slice)
print(fruits)       # affiche : ["pomme"]

del fruits          # supprime carrément la variable "fruits"
# print(fruits)     # ← provoquerait une NameError : fruits n'existe plus

del ne retourne rien. C'est un outil de "suppression pure", sans récupération.

❌

Après del fruits, la variable n'existe plus. Toute tentative d'y accéder provoque NameError: name 'fruits' is not defined.

📖 Dictionnaires et .get()

Le problème : la KeyError

Un dictionnaire, c'est comme un carnet d'adresses : tu donnes un nom (la clé), tu récupères un numéro (la valeur). Mais si le nom n'est pas dans le carnet et que tu accèdes directement avec [], Python plante avec une KeyError.

Python
personne = {"nom": "Alice", "age": 30}  # dictionnaire avec 2 clés

print(personne["nom"])    # ✅ affiche : Alice
print(personne["ville"])  # ❌ KeyError : la clé "ville" n'existe pas

.get() — accès sécurisé sans risque d'erreur

.get() fait la même chose que [], mais au lieu de planter si la clé manque, il retourne None (ou une valeur par défaut que tu choisis).

Explication : on essaie de lire "ville" qui n'existe pas. Avec .get(), pas d'erreur — on obtient juste None.

Python
personne = {"nom": "Alice", "age": 30}

print(personne.get("nom"))    # affiche : Alice  (clé existante → valeur normale)
print(personne.get("ville"))  # affiche : None   (clé absente → None, pas d'erreur)

.get() avec une valeur par défaut

En passant un deuxième argument, tu décides ce que .get() renvoie si la clé n'existe pas.

Python
personne = {"nom": "Alice", "age": 30}

print(personne.get("ville", "inconnue"))  # affiche : inconnue  (valeur par défaut)
print(personne.get("age", 0))             # affiche : 30  (clé existe → valeur réelle)

La valeur par défaut n'est utilisée QUE si la clé est absente.

Cas pratique : compter des occurrences

C'est un usage très fréquent. On veut compter combien de fois chaque mot apparaît dans une liste.

Explication : pour chaque animal, on lit le compteur actuel avec .get(animal, 0) — si l'animal n'a jamais été vu, .get retourne 0. On ajoute 1 et on enregistre.

Python
texte = ["chat", "chien", "chat", "oiseau", "chat"]  # liste d'animaux
compteur = {}                                          # dictionnaire vide pour compter

for animal in texte:                                   # on parcourt chaque animal
    compteur[animal] = compteur.get(animal, 0) + 1    # on incrémente son compteur
    # Si "chat" n'a pas encore de clé → .get retourne 0, puis on fait 0+1=1
    # Si "chat" existe déjà → .get retourne la valeur actuelle, puis on ajoute 1

print(compteur)  # affiche : {"chat": 3, "chien": 1, "oiseau": 1}

Résultat : un compteur d'occurrences construit en 3 lignes, sans aucun if compliqué.

Tableau récapitulatif des accès dictionnaire

SyntaxeClé existanteClé absente
dict["clé"]retourne la valeur❌ KeyError
dict.get("clé")retourne la valeurretourne None
dict.get("clé", "défaut")retourne la valeurretourne "défaut"
🧠

Règle simple : si tu n'es pas certain qu'une clé existe dans ton dictionnaire, utilise toujours .get() plutôt que [].

🔀 Structures de contrôle

Les structures de contrôle, c'est le "cerveau" du programme : elles décident quoi faire, quand le faire, et combien de fois.

Opérateurs logiques : and, or, not

Ces opérateurs permettent de combiner plusieurs conditions dans un if.

Python
age = 20
pays = "France"
connecte = False

# and : les DEUX conditions doivent être vraies
if age >= 18 and pays == "France":          # "majeur" ET "en France"
    print("Majeur en France")               # s'exécute seulement si les deux sont vrais

# or : AU MOINS UNE condition vraie suffit
if age == 20 or age == 21:                  # "20 ans" OU "21 ans"
    print("Tu as 20 ou 21 ans")

# not : INVERSE la condition
if not connecte:                            # si connecte est False → not False = True
    print("Veuillez vous connecter")
OpérateurRésultat vrai quand…
andles deux conditions sont vraies
orau moins une condition est vraie
notla condition de départ est fausse

if / elif / else — choisir selon une condition

C'est le "si... sinon si... sinon" du langage naturel, traduit en Python.

Explication : Python teste les conditions de haut en bas et s'arrête dès qu'une est vraie.

Python
note = 75

if note >= 90:          # si la note est 90 ou plus
    print("Excellent")
elif note >= 70:        # sinon, si la note est 70 ou plus
    print("Bien")       # ← cette branche est exécutée pour note=75
elif note >= 50:        # sinon, si la note est 50 ou plus
    print("Passable")
else:                   # sinon (aucune condition précédente n'était vraie)
    print("Insuffisant")

Résultat : Python affiche "Bien" car 75 >= 70 est vrai et c'est la première condition satisfaite.

💡

Un seul bloc if et un seul else par structure, mais autant de elif que tu veux. Dès qu'une branche est vraie, les suivantes sont ignorées.

while — répéter tant qu'une condition est vraie

while continue à tourner aussi longtemps que sa condition est vraie. C'est la boucle "tant que".

Explication : on veut compter de 0 à 4. La boucle tourne tant que compteur < 5. À chaque tour, on affiche la valeur puis on l'incrémente.

Python
compteur = 0          # initialisation : on part de 0

while compteur < 5:   # condition : on continue TANT QUE compteur est inférieur à 5
    print(compteur)   # affiche la valeur actuelle
    compteur += 1     # CRUCIAL : on incrémente, sinon boucle infinie !

# Affiche : 0, 1, 2, 3, 4

Après le code : sans compteur += 1, la condition compteur < 5 resterait toujours vraie et le programme tournerait indéfiniment. C'est une boucle infinie — à éviter absolument.

⚠️

La boucle infinie est le piège classique de while. Assure-toi toujours que quelque chose dans ta boucle modifie la condition pour qu'elle devienne fausse à un moment.

for — répéter pour chaque élément

for parcourt une collection (liste, chaîne, dictionnaire…) et exécute le code pour chaque élément. Pas besoin de gérer un compteur manuellement.

Python
fruits = ["pomme", "banane", "cerise"]   # liste de 3 fruits

for fruit in fruits:    # pour chaque élément de fruits, on l'appelle "fruit"
    print(fruit)        # affiche l'élément courant

# Affiche : pomme / banane / cerise (un par ligne)

for fonctionne sur tout ce qui est "itérable" : listes, chaînes de caractères, dictionnaires...

Python
for lettre in "Bonjour":              # on parcourt chaque caractère de la chaîne
    print(lettre)                     # B, o, n, j, o, u, r

for cle, valeur in {"a": 1, "b": 2}.items():  # on parcourt les paires clé/valeur
    print(cle, valeur)                # a 1  /  b 2

range() — générer une séquence de nombres

range() crée une suite de nombres, très souvent utilisée avec for pour répéter quelque chose N fois.

Python
# range(stop)           → de 0 à stop-1
# range(start, stop)    → de start à stop-1
# range(start, stop, step) → de start à stop-1, par pas de step

for i in range(5):          # i prend les valeurs 0, 1, 2, 3, 4
    print(i)

for i in range(2, 8):       # i prend les valeurs 2, 3, 4, 5, 6, 7
    print(i)

for i in range(0, 10, 2):   # i prend les valeurs 0, 2, 4, 6, 8 (pas de 2)
    print(i)

for _ in range(5):          # _ signifie "je n'utilise pas le compteur"
    print("Hello")          # affiche "Hello" 5 fois
💡

Le _ dans for _ in range(5) est une convention Python : ça dit "je veux répéter 5 fois, mais je n'ai pas besoin du numéro du tour".

break et continue — contrôler le déroulement de la boucle

Ces deux mots-clés modifient le comportement d'une boucle en cours d'exécution.

Explication : break sort complètement de la boucle, continue passe directement au tour suivant en sautant le reste du code du tour actuel.

Python
# break : quitte la boucle immédiatement quand i vaut 5
for i in range(10):         # i va de 0 à 9
    if i == 5:
        break               # on sort de la boucle, on n'affiche pas 5, 6, 7, 8, 9
    print(i)                # affiche : 0, 1, 2, 3, 4

# continue : saute le tour actuel quand i vaut 3
for i in range(6):          # i va de 0 à 5
    if i == 3:
        continue            # on saute ce tour, on n'affiche pas 3
    print(i)                # affiche : 0, 1, 2, 4, 5
Mot-cléEffet
breaksort complètement de la boucle
continuepasse au tour suivant (ignore le reste du tour)

🛡️ Gestion des erreurs avec try / except

Pourquoi gérer les erreurs ?

Quand Python rencontre une erreur (diviser par zéro, lire un fichier qui n'existe pas, convertir du texte en nombre), il "lève une exception" et le programme s'arrête brutalement. try/except permet d'attraper ces erreurs et de décider quoi faire au lieu de laisser planter.

C'est comme le filet sous un trapéziste : il n'empêche pas la chute, mais il évite la catastrophe.

Structure de base

Explication : le code "à risque" va dans le bloc try. Si une erreur survient, Python saute dans le bloc except correspondant au lieu de planter.

Python
try:
    resultat = 10 / 0           # ← code risqué : division par zéro !
except ZeroDivisionError:       # on attrape ce type d'erreur spécifique
    print("Impossible de diviser par zéro")  # on gère proprement

Résultat : au lieu d'un crash, le programme affiche le message et continue.

Attraper plusieurs types d'erreurs

Python
try:
    valeur = int("abc")         # essaie de convertir "abc" en entier → ValueError
    resultat = 10 / valeur      # si valeur vaut 0 → ZeroDivisionError
except TypeError:               # si le type est incorrect
    print("Mauvais type de données")
except ValueError:              # si la valeur est incorrecte (ex: "abc" → int)
    print("Valeur invalide")
except ZeroDivisionError:       # si on divise par zéro
    print("Division par zéro impossible")
except Exception as e:          # filet général : attrape tout le reste
    print(f"Erreur inattendue : {e}")  # e contient le message d'erreur
🧠

except Exception as e est le "attrape-tout". Utile comme filet de sécurité, mais préfère toujours attraper des erreurs spécifiques en premier — c'est plus lisible et plus précis.

else et finally

Python
try:
    resultat = 10 / 2           # pas d'erreur ici
except ZeroDivisionError:
    print("Division par zéro")
else:
    print("Succès :", resultat) # ← s'exécute SEULEMENT si aucune erreur n'a eu lieu
finally:
    print("Fin du bloc try")    # ← s'exécute TOUJOURS, erreur ou pas
BlocS'exécute quand…
trytoujours (c'est le code à protéger)
exceptseulement si une erreur du bon type survient
elseseulement si aucune erreur n'a eu lieu
finallytoujours, erreur ou pas (utile pour fermer un fichier, par exemple)

🎲 FizzBuzz — Exemple qui combine tout

FizzBuzz est un exercice classique : pour chaque nombre de 1 à 20, on affiche "FizzBuzz" s'il est divisible par 15, "Fizz" s'il est divisible par 3, "Buzz" s'il est divisible par 5, sinon le nombre lui-même.

Le piège est l'ordre des conditions. Si on teste i % 3 == 0 en premier, les multiples de 15 (qui sont aussi multiples de 3) seront capturés par la première condition et n'afficheront jamais "FizzBuzz".

Explication : il faut toujours tester le cas le plus spécifique (multiple de 15) en premier, puis les cas plus généraux.

Python
for i in range(1, 21):          # on parcourt les nombres de 1 à 20 inclus
    if i % 15 == 0:             # cas le plus spécifique EN PREMIER : divisible par 3 ET par 5
        print("FizzBuzz")
    elif i % 3 == 0:            # sinon : divisible par 3 seulement
        print("Fizz")
    elif i % 5 == 0:            # sinon : divisible par 5 seulement
        print("Buzz")
    else:                       # sinon : aucun des cas précédents
        print(i)

L'opérateur % (modulo) donne le reste de la division. 15 % 3 == 0 signifie que 15 est divisible par 3 sans reste.

✅

FizzBuzz combine for, range(), if/elif/else et l'opérateur modulo %. C'est un excellent exercice de consolidation et il est souvent posé en entretien technique.

❌

Erreur classique de FizzBuzz : tester i % 3 == 0 avant i % 15 == 0. Le nombre 15 est divisible par 3, donc la première condition serait vraie et Python n'irait jamais vérifier i % 15 == 0.

🎯 Questions de vérification

  1. Quelle est la différence entre lst.pop(2) et lst.remove("banane") ? Dans quel cas utiliser l'un plutôt que l'autre ?
  1. Que se passe-t-il si tu utilises dict["cle"] avec une clé absente ? Comment .get() résout-il ce problème ?
  1. Explique en une phrase la différence entre while et for. Donne un exemple de situation où tu choisirais while plutôt que for.
  1. Dans FizzBuzz, pourquoi est-il crucial de tester i % 15 == 0 avant i % 3 == 0 ? Que se passerait-il sinon ?

📌 TL;DR

  • pop(index) supprime et retourne l'élément à un index donné ; remove(valeur) supprime la première occurrence d'une valeur (sans retour) ; del est un mot-clé qui supprime par index, slice, ou même une variable entière.
  • .get("clé") lit un dictionnaire sans risque de KeyError ; .get("clé", "défaut") renvoie une valeur par défaut si la clé est absente.
  • and, or, not permettent de combiner des conditions logiques.
  • while tourne tant qu'une condition est vraie — ne pas oublier de modifier la condition sinon boucle infinie.
  • for ... in ... parcourt chaque élément d'une liste, chaîne ou dictionnaire.
  • range(start, stop, step) génère une suite de nombres utilisable avec for.
  • break sort d'une boucle immédiatement ; continue passe au tour suivant.
  • try/except capture les erreurs et évite les plantages ; else s'exécute si tout s'est bien passé ; finally s'exécute toujours.
  • Dans FizzBuzz, tester toujours le cas le plus spécifique (% 15) avant les cas généraux (% 3, % 5).

A. Concepts du cours

Le bon usage de la documentation

La compétence numéro un d'un développeur, c'est de savoir lire la documentation officielle. Pas Stack Overflow, pas un blog tutoriel : la doc officielle.

Pourquoi ? Parce qu'elle est :

  • Exhaustive — tous les paramètres sont listés
  • À jour — elle reflète la version actuelle de la lib
  • Précise — elle décrit exactement le comportement, pas une approximation

Stack Overflow contient souvent des réponses obsolètes (Python 2 vs 3, sklearn 0.18 vs 1.4) qui « marchent presque » et qui vous coûteront trois heures de debug.

Trois sources canoniques pour Python data science :

Python
# 1. help() : docstring dans le REPL
help(pd.DataFrame.merge)

# 2. Version courte IPython/Jupyter (affiche la signature + docstring)
pd.DataFrame.merge?

# 3. Inclut le code source de la fonction
pd.DataFrame.merge??

Exemple concret : vous cherchez comment faire un left join avec pandas.

  • Mauvais réflexe : googler "pandas left join" → arriver sur une réponse Stack Overflow de 2015 qui utilise ignore_index=True (obsolète).
  • Bon réflexe : pd.DataFrame.merge? → voir how='left' et comprendre les 4 options en une phrase.
💡

Analogie : Stack Overflow est à la doc officielle ce qu'est une discussion de comptoir au manuel d'un médicament. Les deux ont leur place, mais avant de prendre un risque, on lit la notice. En data science, la "prise de risque" c'est lancer un entraînement de 8 h basé sur une mauvaise compréhension d'un paramètre.

Lire un traceback Python

Quand un script plante, Python affiche un traceback : la pile d'appels qui a mené à l'erreur. Un débutant lit la dernière ligne (le type d'erreur) et tente une correction au hasard. Un pro lit le traceback de bas en haut :

  1. La toute dernière ligne donne le type d'erreur (KeyError, ValueError, TypeError) et son message.
  2. La ligne juste au-dessus indique le fichier et la ligne où ça plante.
  3. Au-dessus encore, on remonte la chaîne d'appels jusqu'à VOTRE code (souvent enfoui sous des frames de bibliothèques).
Code
Traceback (most recent call last):
  File "train.py", line 42, in <module>
    model.fit(X_train, y_train)
  File ".../sklearn/linear_model/_base.py", line 651, in fit
    X, y = self._validate_data(X, y, ...)
ValueError: could not convert string to float: 'unknown'

Diagnostic :

  • Type : ValueError — conversion impossible.
  • Message : could not convert string to float: 'unknown' — quelque part, la chaîne 'unknown' devait être un nombre.
  • Où : sklearn/.../_base.py ligne 651 — c'est sklearn qui plante, mais le VRAI bug est dans VOS données.
  • Action : retourner à train.py ligne 42, inspecter X_train. Probablement une colonne catégorielle mal encodée.

Le diagnostic n'est PAS « il y a un bug dans sklearn », c'est : X_train contient une chaîne 'unknown' quelque part. Le fix est dans votre preprocessing, pas dans le modèle.

Python
# Avant de relancer, investiguer :
print(X_train.dtypes)         # Quelle colonne est 'object' alors qu'elle devrait être numérique ?
print(X_train.isin(['unknown']).any())  # Quelle colonne contient 'unknown' ?
⚠️

Piège fréquent : KeyError: 'date' ne signifie pas toujours que la clé 'date' manque. Si vous faites df['date'] après un df.groupby('date').sum(), la colonne date est devenue l'index et n'est plus une colonne. Solution : df.reset_index() ou utiliser df.index directement.

Conventions de nommage

Python a des conventions PEP 8 strictes que les recruteurs scrutent :

TypeConventionExemple
Variable / fonctionsnake_casetrain_score, compute_loss()
ConstanteUPPER_SNAKEMAX_EPOCHS = 100
ClassePascalCaseclass RandomForestClassifier:
Privépréfixe __internal_state, _helper()
Nom "magique"__double_underscore____init__, __repr__

Les noms doivent être explicites mais pas verbeux. df_clean est meilleur que df1 ou data_after_cleaning_step_2. La règle de Kent Beck : « plus la portée d'une variable est large, plus son nom doit être long ».

Exemples :

Python
# ❌ Trop court, portée large, on ne sait pas ce que c'est
def process(d):
    for i in d:
        ...

# ✓ Explicite
def clean_transactions(transactions):
    for transaction in transactions:
        ...

# ✓ Court OK pour une variable locale à portée minimale
for i in range(10):     # i = indice de boucle, convention universelle
    print(i)
👉

Bonne pratique : aucune variable d'une seule lettre sauf dans les boucles (for i in range(n)) et les contextes mathématiques où la lettre est canonique (X, y, theta, alpha). Pour for x in df, préférez for row in df — c'est plus parlant et évite la confusion avec un X matriciel.

B. Compléments avancés

Profilage rapide d'un script lent

Avant d'optimiser, mesurez. La règle d'Amdahl : « optimisez d'abord ce qui prend le plus de temps ».

Niveau 1 — timing manuel (pour une mesure ponctuelle) :

Python
import time
t0 = time.perf_counter()   # perf_counter : horloge monotone haute précision
# ... votre code ...
print(f"Took {time.perf_counter() - t0:.2f}s")
# Result: Took 3.42s

Niveau 2 — découpage par fonction (cProfile) :

Python
import cProfile, pstats

profiler = cProfile.Profile()
profiler.enable()
main()                                                      # votre fonction principale
profiler.disable()
pstats.Stats(profiler).sort_stats('cumulative').print_stats(20)
# Affiche les 20 appels les plus coûteux (en temps cumulatif)

Niveau 3 — magic commands Jupyter :

Python
%timeit my_function(x)         # Mesure répétée + statistiques (moyenne, écart-type)
%prun -l 10 my_function(x)     # Profilage avec top 10

Niveau 4 — profilage mémoire :

Python
from memory_profiler import profile

@profile
def heavy_function():
    data = [i * 2 for i in range(10**7)]
    return sum(data)
# Lancer avec : python -m memory_profiler script.py
# Affiche la mémoire allouée ligne par ligne

Gérer les imports proprement

PEP 8 impose un ordre strict des imports :

Python
# 1. Bibliothèque standard
import os
import sys
from pathlib import Path

# 2. Bibliothèques tierces
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split

# 3. Modules locaux (votre projet)
from src.preprocessing import clean_data
from src.models import build_model

Trois règles à mémoriser :

  1. Pas d'import * — pollue le namespace, casse les linters, l'IDE n'autocomplète plus.
  2. Pas d'imports dans les fonctions (sauf cas exceptionnels : import lent, dépendance optionnelle).
  3. Trier alphabétiquement dans chaque groupe. isort le fait automatiquement.
⚠️

Piège : faire from numpy import * puis utiliser min(x) retourne en fait numpy.min, qui se comporte différemment du builtin Python sur les listes vides ou les générateurs. Bug invisible jusqu'au jour où ça plante en production avec un message incompréhensible.

Mode interactif et REPL ipdb

pdb (le debugger Python) est un super-pouvoir sous-estimé. Au lieu d'ajouter dix print() puis de les enlever, posez un breakpoint :

Python
def compute_score(y_true, y_pred):
    breakpoint()              # Python 3.7+, équivaut à import pdb; pdb.set_trace()
    return f1_score(y_true, y_pred)

Quand le code atteint cette ligne, vous tombez dans un REPL interactif où vous pouvez inspecter toutes les variables, exécuter du code arbitraire, et avancer pas à pas.

Commandes essentielles :

CommandeAction
n (next)Exécute la ligne courante (reste dans la même fonction)
s (step)Entre dans la fonction appelée
c (continue)Reprend l'exécution jusqu'au prochain breakpoint
l (list)Affiche le contexte de code
p varImprime la variable
pp varPretty-print (utile pour les dicts/listes imbriqués)
q (quit)Quitte le debugger (et le programme)

ipdb est la version améliorée avec coloration syntaxique et autocomplétion : pip install ipdb puis import ipdb; ipdb.set_trace().

💡

Analogie : print() c'est observer un poisson dans son aquarium depuis l'extérieur. pdb c'est mettre une combinaison de plongée et nager avec lui. Pour des bugs subtils (mauvaise valeur, état imprévu), pdb vous fera économiser des heures.

C. Questions de compréhension

Q1 : Vous avez une erreur ImportError: No module named 'sklearn' mais vous avez pourtant fait pip install scikit-learn. Quelle est probablement la cause ?

💡 Voir l'indice

Environnements multiples Python.

✅ Voir la réponse

Le pip que vous appelez n'est pas associé au python qui exécute votre script. C'est le piège le plus classique : vous avez plusieurs Python installés (system, brew, pyenv, conda, venv) et pip install a installé dans l'un, tandis que votre script s'exécute dans un autre.

Diagnostic :

Bash
which python3   # Ex: /Users/alice/.pyenv/shims/python3
which pip3      # Ex: /usr/local/bin/pip3     ← DIFFÉRENT !

Solution canonique :

Bash
python3 -m pip install scikit-learn
# Le -m garantit que c'est CE python3 qui installe,
# donc l'install va dans le bon site-packages

Pour éviter complètement ce problème, toujours travailler dans un environnement virtuel dédié au projet :

Bash
python3 -m venv .venv         # Crée un venv local au projet
source .venv/bin/activate     # Active (modifie PATH)
pip install scikit-learn      # Installe dans le venv local

Désormais python et pip sont garantis d'être alignés.

Q2 : Quelle est la différence entre is et == en Python, et pourquoi a = 256; b = 256; a is b retourne True mais a = 257; b = 257; a is b retourne souvent False ?

💡 Voir l'indice

Interning des petits entiers.

✅ Voir la réponse
  • == compare les valeurs (appelle __eq__).
  • is compare les identités — même objet en mémoire, c'est-à-dire id(a) == id(b).

Pour les petits entiers (-5 à 256), CPython les pré-alloue au démarrage et les réutilise — c'est l'interning. Donc a = 256 et b = 256 pointent vers le même objet en mémoire. Pour 257, chacun crée son propre int distinct.

Python
a = 256; b = 256
a is b   # True (interning)

a = 257; b = 257
a is b   # False dans un script, parfois True dans un REPL (optimisation de compilation)

Règle absolue : utilisez is uniquement pour comparer à None, True, False ou pour vérifier l'identité d'objets (rare). Pour comparer des valeurs, toujours ==.

Python
if x is None:     # ✓ Convention Python
if x == None:     # ❌ Fonctionne mais non idiomatique, plus lent

if user.id == 42: # ✓ Comparaison de valeur
if user.id is 42: # ❌ Peut marcher par hasard via interning, bug invisible

C'est une question piège classique en entretien Python pour éliminer ceux qui ont survolé la doc.

Q3 : Vous lancez un script qui marche sur 1000 lignes mais consomme 8 Go quand vous le lancez sur 1 million de lignes. Avant tout refactoring, quelle est la première mesure à prendre ?

💡 Voir l'indice

Ne pas optimiser à l'aveugle — profiler d'abord.

✅ Voir la réponse

Mesurer où la mémoire est consommée. tracemalloc (stdlib) ou memory_profiler (avec @profile) vous diront ligne par ligne quelle allocation explose.

Python
import tracemalloc

tracemalloc.start()
main()
current, peak = tracemalloc.get_traced_memory()
print(f"Current: {current / 1e9:.2f} GB, Peak: {peak / 1e9:.2f} GB")
tracemalloc.stop()

La cause est presque toujours l'une de trois :

  1. df.copy() implicite à chaque opération qui duplique le DataFrame. Chaque df[df.x > 0] peut copier le df entier selon Pandas.
  2. Concaténation dans une boucle : pd.concat([df, new_row]) à chaque itération est O(n²) en mémoire et en temps. Solution : accumuler dans une liste puis pd.concat une seule fois à la fin.
  3. Lecture intégrale au lieu d'un streaming par chunks : pd.read_csv(..., chunksize=10000) au lieu de tout charger.

Le mauvais réflexe est de jeter le code et de tout réécrire en Polars/Dask. Le bon réflexe est d'identifier la ligne problématique et de la corriger avec une opération in-place ou un chunking. Règle : profiler, comprendre, corriger localement, dans cet ordre.
